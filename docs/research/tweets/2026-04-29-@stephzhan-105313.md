## 全文

4/ Hiring has not caught up.

If you want to hire agentic engineers, toy puzzles are the wrong test.

Give candidates a real project. Let them use the tools. Make them deploy it. Then try to break it with agents.

The question is no longer just: can you solve the problem?

It's: can you ship a system that survives contact with reality?

---

## 研究摘要

**TL;DR**：Agentic engineer 的面試標準應是「能否 ship 一個能在現實中存活的系統」，而非傳統 LeetCode 題。

### 核心論點

- 傳統 toy puzzle 面試（LeetCode/算法題）無法評估 agentic engineering 能力
- 正確評估方法：給真實專案 → 讓用工具 → 部署 → 用 agent 破壞測試
- 評估核心問題從「能解決問題嗎？」→「能 ship 一個能生存的系統嗎？」

### 關鍵數據 / 工具

- 測試流程：real project → use tools → deploy → break it with agents（4 步驟）

### 批判性觀察

- 「用 agent 破壞測試」是有創意的評估框架；對應 cc-workspace 的 healthcheck.sh + deep-review 就是對 agent 輸出的「破壞測試」
- 實際面試中執行難度高：需要時間、環境、評估標準，hiring 成本會顯著上升
