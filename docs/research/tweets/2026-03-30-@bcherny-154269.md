## 全文

3/ Two of the most powerful features in Claude Code: /loop and /schedule

Use these to schedule Claude to run automatically at a set interval, for up to a week at a time.

I have a bunch of loops running locally:

- /loop 5m /babysit, to auto-address code review, auto-rebase, and shepherd my PRs to production
- /loop 30m /slack-feedback, to automatically put up PRs for Slack feedback every 30 mins
- /loop /post-merge-sweeper to put up PRs to address code review comments I missed
- /loop 1h /pr-pruner to close out stale and no longer necessary PRs
- lots more!..

Experiment with turning workflows into skills + loops. It's powerful.

https://code.claude.com/docs/en/scheduled-tasks

## 研究摘要

**TL;DR**：Claude Code 作者親身展示 /loop + /schedule 組合，將 PR 守護、Slack 回饋收集等重複性 PR 工作流轉為定時自動執行迴圈。

### 核心論點

1. `/loop` 與 `/schedule` 是 Claude Code 的「最強兩個功能」——作者語氣強烈，非一般評論
2. `/loop` 可設定間隔（5m / 30m / 1h）無限循環執行任一 Skill，最長持續一週
3. 實際運行的 4 個迴圈展示了從 PR 監護（/babysit）、Slack 回饋 PR 化（/slack-feedback）、post-merge 掃尾（/post-merge-sweeper）到廢 PR 清理（/pr-pruner）的完整生命週期自動化
4. 核心哲學：**把工作流封裝成 Skill，再用 Loop 驅動它**——Skill 是原子操作，Loop 是排程引擎
5. 這是 thread 第 3 則（3/），說明 Boris 在這個 thread 系統性分享 Claude Code 隱藏/進階功能

### 關鍵數據 / 工具

- `/loop 5m /babysit`：每 5 分鐘執行，PR 自動 rebase 與 code review 回應
- `/loop 30m /slack-feedback`：每 30 分鐘自動化 Slack 回饋 → PR 轉換
- `/loop /post-merge-sweeper`：merge 後遺漏 review comment 補處理
- `/loop 1h /pr-pruner`：每小時清理 stale PR
- 官方文件：https://code.claude.com/docs/en/scheduled-tasks
- 最長執行上限：一週

### 批判性觀察

1. 這 4 個迴圈全部依賴 /babysit、/slack-feedback 等**自定義 Skill**，但推文未展示這些 Skill 的內容——可行動性需要使用者自行設計對應 Skill，門檻不低
2. `/loop` 在 cc-workspace 的 `routines.md` 已記載，但缺少 Boris 親身驗證的具體 loop 配方——此推文補充了真實使用案例，是對 routines.md 的高品質補充
3. 在無本機 terminal 常駐的情境下 /loop 會中斷（vs /schedule 雲端執行不需 terminal），推文未說明兩者差異，可能誤導使用者選錯工具
