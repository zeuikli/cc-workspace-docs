## 全文

### Trending: 12 hooks, 51 MCP tools, and a triple-stream retrieval pipeline that scores 95.2% R@5 on LongMemEval-S

agentmemory replaces CLAUDE.md, .cursorrules, and every other static-file memory hack with an actual local service.

3,000+ stars in ~3 days, version 0.9.9 shipped 2026-05-11, Apache-2.0, one npx command to install.

@ghumare64, Principal Product Evangelist at iii.dev, authored the implementation. The design comes from his gist "LLM Wiki v2" which extends Karpathy's context engineering framework.

**What changes for you.** Session 1, you set up JWT auth with jose middleware in src/middleware/auth.ts. Session 2, you ask for rate limiting, and the agent already knows which middleware stack you're using.

## How It Works

agentmemory is built on iii-engine, a service composition framework (15K+ stars, TypeScript + Rust). The system has three layers: Capture, Pipeline, Retrieval. Plus a consolidation cycle.

- **Capture**: 12 Claude Code lifecycle hooks fire automatically — SessionStart, UserPromptSubmit, PreToolUse, PostToolUse, PostToolUseFailure, PreCompact, and more
- **Pipeline**: SHA-256 dedup catches repeats within 5 minutes. Privacy filter removes PII. Importance classifier scores and routes observations.
- **Retrieval**: Three streams run in parallel. BM25 (stems with Porter, synonym expansion). Vector (cosine similarity via embeddings). Graph (entity relationship traversal). Hybrid scoring combines all three — 95.2% R@5 on LongMemEval-S.
- **4-tier consolidation**: Working → Episodic → Semantic → Procedural — analogous to sleep consolidation
- **MCP surface**: 51 memory_* tools, 6 MCP resources, 3 prompts, 4 skills (/recall, /remember, /session-history, /forget), 127 REST endpoints

## How to Get Started

Total time to working install: under 10 minutes on macOS or Linux.

1. **Prerequisites**: Node.js >= 20.0.0, iii-engine v0.11.2 OR Docker Desktop
2. **Start the server**: `npx @agentmemory/mcp` (auto-downloads iii-engine v0.11.2 — do NOT use v0.11.6, it introduced sandbox-per-worker model agentmemory hasn't refactored to yet)
3. **Seed and verify**: Run the demo script to seed 3 sessions (JWT auth, N+1 query fix, rate limiting) and verify search works
4. **Open viewer**: Live observation stream, session explorer, memory browser, knowledge-graph visualization, health dashboard (bound to 127.0.0.1 only)
5. **Wire to agent**: One JSON block covers Cursor, Claude Desktop, Cline, Roo Code, Windsurf, Gemini CLI, OpenClaw
6. **Claude Code**: Install the plugin — registers all 12 hooks, 4 skills, auto-wires MCP server through .mcp.json

Optional: free local embeddings with all-MiniLM-L6-v2 (+9 percentage points recall over BM25-only, no API key).

## Benchmark Evidence

LongMemEval-S (ICLR 2025, 500 questions, ~48 sessions per question, ~115K tokens each):
- Retrieval recall R@5: 95.2% (agentmemory) vs. 78.4% (BM25-only) vs. 71.3% (vector-only)

These are **retrieval recall scores, not end-to-end QA accuracy**. The repo explicitly does not claim these as "LongMemEval scores."

Token efficiency: ~1,900 tokens per session, ~170K per year, ~$10/year on per-token billing or $0/year with local embeddings.

## Current Limitations (12 May)

- **Plaintext HTTP token transport** (issue #275, open): plugin sends AGENTMEMORY_SECRET bearer over plaintext HTTP; localhost-only mitigates but doesn't eliminate risk
- **Interesting features off by default**: auto-compress, context injection, slots, reflect, graph extraction all gated behind env vars
- **Documentation drift**: AGENTS.md ships v0.8.9 stats (44 tools, 104 endpoints); source has 51 tools, 127 endpoints
- **Engine writes state into project root** (issue #303, open): when Claude Code auto-starts from a project directory, iii-engine writes state there

## AlphaSignal Take

**Verdict: Production Ready for solo developers and small teams** running agentmemory on localhost as a personal coding-agent memory layer.

The architecture is real. The benchmark is reproducible from committed scripts. The install is one npx command.

What would change the verdict positively: Q4 2026 ships external security audit, SSO, RBAC, audit-log export. Q1 2027 freezes REST and MCP API surface.

**Benefits**: Claude Code, Cursor, Codex CLI, Gemini CLI users running solo or in small teams.  
**Doesn't benefit yet**: teams deploying beyond localhost without reverse proxy + TLS.

## 研究摘要

**TL;DR**：agentmemory 用三流檢索管道（BM25+向量+圖）取代靜態 CLAUDE.md，95.2% R@5 召回率，一行安裝

### 核心論點

1. **三層架構**：Capture（12 hooks）→ Pipeline（SHA-256去重、PII過濾、重要性分類）→ Retrieval（BM25+向量+圖混合）
2. **4 層記憶鞏固**：類睡眠鞏固機制，Working → Episodic → Semantic → Procedural
3. **技術基礎**：iii-engine（TypeScript + Rust），51 個 MCP 工具，127 個 REST endpoints
4. **已知風險**：HTTP 明文傳輸 token（issue #275），Engine 在專案根目錄寫入狀態（issue #303）
5. **版本注意**：僅支援 iii-engine v0.11.2，不要用 v0.11.6

### 關鍵數據 / 工具

- 評測：LongMemEval-S，500 題，R@5 = 95.2%（vs. BM25-only 78.4%，vector-only 71.3%）
- Token 效率：~1,900 tokens/session，~$10/年（API 計費）或 $0/年（本地嵌入）
- 嵌入模型：all-MiniLM-L6-v2（本地，免費，+9% recall）
- Repo：github.com/rohitg00/agentmemory，3,000+ stars in ~3 days
- 版本：0.9.9（2026-05-11），Apache-2.0

### 批判性觀察

1. **召回率 ≠ QA 準確率**：文章誠實標注，但 R@5 95.2% 對實際效果的轉化率未見說明
2. **未成熟的安全模型**：issue #275（明文 token）和 #303（專案根目錄污染）是生產環境真實風險
3. **文件落後於代碼**：v0.8.9 文件 vs v0.9.9 代碼，說明項目迭代快但文件維護跟不上
