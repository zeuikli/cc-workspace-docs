## 全文

每天出新东西，但关上手机那一刻什么都没进脑子。

三步工作流：**換高質量信息源 → 讓 Hermes Agent 每天替你篩選消化 → 倒逼高質量輸出**

### 高品質 AI Newsletter 清單（50+ 篩選後）

- The Rundown AI（therundown.ai）：每日 AI 産業動態
- TLDR AI（tldr.tech）：5 分鐘科技简报
- The Neuron（theneurondaily.com）：非技術讀者日報
- Ben's Bites（bensbites.com）：Ben Tossell AI 日報
- Latent Space（latent.space）：swyx 開發者简报，未公開項目+工程視角
- Smol AI News（smol.ai/news）：Discord、Reddit AI 社區直擊
- Interconnects（interconnects.ai）：Nathan Lambert RLHF/模型訓練深度
- One Useful Thing（oneusefulthing.org）：Ethan Mollick AI 實用洞察
- AI Breakfast（aibreakfast.beehiiv.com）：每週 AI 新聞早餐
- Every（every.to）：科技與商業戰略分析

### 構建管道：4 步設置

**第一步**：給 Hermes 配置邮箱（ClawEmail — 網易邮箱的 Agent 專屬邮箱）

**第二步**：裝 `mail-cli` 讓 Hermes 有「眼睛」（可讀邮件、搜關鍵詞、拉正文）

**第三步**：建 newsletter 專屬子邮箱（隔離信息，提升 Agent 處理效率）

```
Dashboard → Agent 邮箱管理 → 通讯規則 → 開放外部通信 → 收信範圍：所有人
```

**第四步**：讓 Agent 訂閱 newsletters（確認邮件由 Hermes 處理）

### 筛選消化：Hermes 構建第二大腦

**第一步**：掃 Digest（ClawEmail 自帶，每天一封摘要日報，選出最值得深讀的文章）

**第二步**：LLM Wiki 技術消化知識

LLM Wiki 概念：像個實習生，每天把文章塞進去，它把知識從「一篇文章的形式」轉化成「能被查詢的知識點」存入個人 wiki。當你寫作時問「上週哪幾篇文章支持這個觀點」，它能直接回答。

**第三步**：倒逼輸出，用 Hermes 協助寫作

寫作時問 Hermes：
1. 這個概念我上週讀過哪幾篇支持？
2. 那個觀點我怎麼反駁？
3. 還有哪些角度我沒考慮到？

## 研究摘要

**TL;DR**：用 Hermes Agent + Agent 專屬邮箱（ClawEmail）自動化 AI Newsletter 訂閱-消化-輸出工作流，核心是 LLM Wiki 技術將文章轉化為可查詢知識庫。

### 核心論點

- 信息攝入問題的根源：學 AI 但知識沒有進入長期記憶，需要主動的消化和輸出環節
- LLM Wiki 概念：文章 → 知識點化 → 可查詢個人 wiki，形成真正的第二大腦
- Agent 專屬邮箱（ClawEmail）是構建信息管道的基礎設施
- 輸出倒逼輸入：以「寫出能打的文章」為目標驅動信息攝取

### 關鍵數據 / 工具

- 信息源：從 50+ 篩選出 10 個頂級 newsletter（含 Latent Space、Interconnects 等工程向）
- ClawEmail：網易邮箱的 Agent 專屬邮箱（內測，邀請碼 CLAW3C02D3891B6C，限 200 次）
- mail-cli：讓 Agent 可讀邮件的 CLI 工具
- 作者成果：連續產出 18 萬、17 萬、86 萬閱讀的文章

### 批判性觀察

- 文章含明顯推廣成分（ClawEmail 邀請碼、網易邮箱 hashtag），中立性存疑
- LLM Wiki 技術描述模糊，未說明具體實現方式（是 RAG？是 structured memory？）
- Newsletter 清單本身有高價值（Latent Space、Interconnects 是工程師 community 的重要信號源）
