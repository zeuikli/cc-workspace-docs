## TL;DR

實作時要求 AI 同時維護 `implementation-notes.html`，自動記錄所有 spec 未涵蓋的決策、變更與取捨，讓隱性決策變顯性。

## 核心論點

1. AI 在實作過程中做了大量未被記錄的決策，用戶事後無法追溯
2. 在 prompt 中要求 AI 維護 running notes 檔案，把這些決策即時記下來
3. 捕捉四類資訊：spec 空白決策、被迫變更、取捨點、其他需知事項
4. HTML 或 Markdown 均可，依工作流選擇

## 關鍵 Prompt Template

```
implement <SPEC> and while you do, keep a running implementation-notes.html file (or markdown)
with decisions you had to make weren't in the spec,
things you had to change,
tradeoffs you had to make
or anything else I should know
```
