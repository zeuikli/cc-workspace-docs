### 1️⃣ 上下文管理：別讓「歷史」拖垮你（2037460567485780335）

开启新对话：每一个新消息都会发送全部历史记录。话题变了就赶紧开新坑，或者用 `/clear`。

精简 CLAUDE.md：这个文件是会被持续加载到内存的，越臃肿越烧钱，保持 Lean 状态。

警惕自动记忆：检查 `/memory` 是否开启。开启它有时会导致上下文分叉，产生双倍扣费。

---

### 2️⃣ 提示詞工程：拒絕「無效思考」（2037460571797557501）

精准打击：别说「帮我修下 Bug」，直接说「修复 src/auth.ts 里的中间件逻辑」。搜索范围越小，Token 越省。

慎用 Ultrathink（深度思考）：很多人觉得思考越久效果越好，其实不然。复杂逻辑才用它，普通任务开启它是纯纯的浪费。

降级策略：不是所有活都要 Opus 4.6。非深度逻辑任务，果断选 Sonnet，性价比之王。

---

### 3️⃣ 工具與 MCP：關掉「偷跑」流量（2037460575379427489）

清理 MCP 服务器：哪怕你没主动用，挂着的 MCP 也会占用上下文。不用的 Server 赶紧关掉。

本地预处理：别把几千行的原始测试日志直接扔给 Claude。先用本地小模型（如 Llama 3）或低成本模型总结出关键点，再投喂给 Claude。Token 占用最高可降低 99%！

⚡ 总结一句话：像用流量套餐一样用 Claude：按需开启，用完即关，保持精简。

## 研究摘要

**TL;DR**：Reddit 社群整理的 Claude Token 省量三大策略：上下文管理（精簡 CLAUDE.md、話題切換即 /clear）、提示詞精準化（禁用不必要的 Ultrathink、模型降級）、MCP 清理與本地預處理（可降低 token 消耗最多 99%）。

### 核心論點

1. **Context 是最大浪費源**：每次對話都附帶完整歷史，CLAUDE.md 越長消耗越多；切換話題不開新對話是最常見的 token 浪費行為
2. **Ultrathink 應按需使用**：深度思考模式只適合複雜邏輯，普通任務開啟是純粹浪費
3. **模型降級策略**：Opus 用於深度推理，Sonnet 用於一般任務；選錯模型直接影響成本
4. **MCP 空跑消耗**：即使未主動呼叫，掛載的 MCP Server 仍會佔用上下文 token
5. **本地預處理截斷輸入**：用本地小模型（Llama 3 等）先壓縮長文本再投喂 Claude，可降低 token 消耗高達 99%

### 關鍵數據 / 工具

- 本地預處理節省幅度：最高 99%（文中聲稱，未附測試條件）
- 提及工具：`/clear`、`/memory`、MCP Server、Llama 3（本地預處理）
- 模型分層：Opus（深度邏輯）→ Sonnet（一般任務）

### 批判性觀察

1. **本地預處理 99% 降幅**：聲稱最高降低 99% token 消耗，但未說明基準條件（如何定義「幾千行日誌」），實際效果依任務類型差異大
2. **與 workspace 現有規則高度重疊**：`token-efficiency.md`、`context-management.md` 已涵蓋多數策略，增量價值有限
3. **MCP 空跑說明有工程價值**：「未主動呼叫的 MCP 仍消耗 context」這點在 workspace 規則中未明確點出，是可補充的 gotcha
