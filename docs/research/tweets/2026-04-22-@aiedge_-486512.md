## 全文

I don't care what anyone tells you - by default, Claude's memory is basically useless.

It frequently forgets context; you constantly have to re-explain yourself, and even after you do, it still often doesn't remember.

I discovered three "layers" of memory systems, and they've made Claude significantly more powerful.

#### Layer one: Basic Memory (Beginner)

**1. Memory Editing Tool**

Settings → Memory：管理 Claude 自動累積的記憶（刪除過時/不準確的，手動添加關鍵 context）

**2. Project Instructions**

在 Claude Projects 的 Project Instructions 欄位填入 context。建議：用語音轉文字整理成 Google Doc 再上傳 PDF。

**3. Tell Claude Directly**

直接在對話中告訴 Claude 要記住什麼；也可說「Forget that I mentioned [x]」

**4. Memory Imports & Exports**

Settings → Memory 可 import 來自其他 LLM（ChatGPT 等）的 context。

→ 這 4 個設定滿足 90%+ 用戶

#### Layer two: Context File System (Intermediate)

設置時間：約 60 分鐘。核心概念：將 context 儲存在本機 .MD 檔案，Claude Code/Cowork 自動載入。

**Claude Master Folder 四個 markdown 檔案：**

1. **Instructions.MD** — Claude 的規則和指令
```markdown
## Who you are
## What you do
## Rules
## What good outputs look like
```
關鍵行：「Update Memory.MD with my preferences over time.」

2. **Memory.MD** — Claude 的「大腦」，持續自動更新
```markdown
## Preferences
## Corrections
## Patterns
## Decisions
```
當你說「stop using em dashes」時，Claude 會自動更新這個檔案。

3. **Context.MD** — 特定專案的 context 檔案

4. **Archive Copies** — 每週手動備份整個 master folder 到 Claude 無法存取的位置，防止意外覆寫

**使用方式**：在 Cowork/Claude Code 中 attach Master Folder → Claude 自動用作 mini memory database，編輯 memory markdown file

## 研究摘要

**TL;DR**：Claude 記憶系統三層架構：Layer 1（原生設定快速修復）→ Layer 2（本機 .MD 檔案作為持久化 context）→ Layer 3（自進化第二大腦），60 分鐘設置的 Context File System 是最高槓桿的中級解決方案。

### 核心論點

- Claude 預設記憶不可用，三層分級解決方案讓 90% 使用者可在 10 分鐘內改善
- Layer 2 的關鍵洞見：Instructions.MD 中的「Update Memory.MD with my preferences over time」一行，讓 Claude 自動維護記憶文件
- Memory.MD 分層（Preferences/Corrections/Patterns/Decisions）是一個實用的 context 組織框架
- Archive 備份策略防止 Claude 自動更新時意外覆寫：每週手動備份到 Claude 無法存取的資料夾

### 關鍵數據 / 工具

- Layer 2 設置時間：約 60 分鐘
- Settings → Memory 匯入/匯出支援跨 LLM 遷移
- 受益用戶群：文章作者稱滿足 90%+ 用戶

### 批判性觀察

- Layer 2 的 Context File System 概念與 cc-workspace 的 CLAUDE.md + Auto Memory 機制高度重疊
- Memory.MD 自動更新機制依賴 Claude 的判斷，可能導致非預期修改（文章本身承認此問題並建議備份）
- Layer 3（AI 第二大腦）的細節在本推文 JSON 中截斷，無法評估
