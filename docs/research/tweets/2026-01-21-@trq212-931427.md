## 全文

Most people's mental model of Claude Code is that "it's just a TUI" but it should really be closer to "a small game engine".

For each frame our pipeline constructs a scene graph with React then
-> layouts elements
-> rasterizes them to a 2d screen
-> diffs that against the previous screen
-> finally uses the diff to generate ANSI sequences to draw

We have a ~16ms frame budget so we have roughly ~5ms to go from the React scene graph to ANSI written.

## 研究摘要

**TL;DR**：Claude Code TUI 是 React 場景圖驅動的遊戲引擎，有 16ms 幀預算，5ms 用於 React→ANSI 路徑。

### 核心論點

- 大眾誤解 Claude Code 是「普通 TUI 工具」，正確心智模型應是「小型遊戲引擎」
- 每幀 pipeline：React 場景圖構建 → 元素 layout → 2D 螢幕光柵化 → 與前幀 diff → 生成 ANSI 序列繪製
- 整體幀預算約 16ms，其中 React 場景圖到 ANSI 輸出的路徑僅有約 5ms
- 這是 Claude Code 核心開發者（Anthropic）首次公開揭露內部渲染架構

### 關鍵數據 / 工具

- 幀預算：~16ms（類 60fps 遊戲標準）
- React→ANSI 路徑預算：~5ms
- 技術棧：React（場景圖）+ 自製 rasterizer + screen diffing + ANSI sequence generator
- Pipeline 5 個階段：scene graph → layout → rasterize → diff → ANSI

### 批判性觀察

- 架構揭露來自 Claude Code 核心工程師，一手資料可信度高，但僅限此推文的單次說明，無程式碼或 benchmark 佐證
- 「16ms 幀預算」說法暗示 Claude Code 以 ~60fps 運行，對純文字 TUI 而言是高規格要求，說明其追求流暢感的工程投入
- 對一般 cc-workspace 使用者無直接可行動性，但對理解 Claude Code 效能限制（例如為何複雜 UI 在慢速終端卡頓）有間接幫助
