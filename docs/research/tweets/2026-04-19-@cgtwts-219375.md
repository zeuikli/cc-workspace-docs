## 研究摘要

**TL;DR**：整理 Claude Code 創作者 Boris Cherny 的工作流程核心原則——平行執行、共享 CLAUDE.md、使用最強模型反而更省錢、給予 feedback loop。

### 核心論點

- 平行運行 5-10 個 Claude 實例（包括從手機）是 Boris 的標準工作模式
- 反直覺的 token 節省邏輯：用最強模型（更貴/token）反而總成本更低，因為它更少失誤、更少重試
- 共享且頻繁更新的 CLAUDE.md 是避免重複錯誤的關鍵機制
- Feedback loop 設計：讓 Claude 跑程式碼並在瀏覽器看結果，而非單純文字交流
- Plan → Lock → Apply 的三步工作流，簡單但高效

### 關鍵數據 / 工具

- 平行實例數：5-10 個 Claude
- 核心工具：CLAUDE.md（共享更新）、Plan mode（鎖定計劃後執行）
- 重要原則：最強模型 → 更少 token 消耗 → 更便宜
- Boris Cherny：Claude Code 創作者

### 批判性觀察

- 「15 個技巧」的標題但推文實際只整理了 3-4 個核心原則，內容與標題不符
- 這些原則在本 workspace 的 CLAUDE.md 和 `.claude/rules/` 中已有大量落地實踐
- 「最強模型更省錢」的論述缺乏量化依據，對複雜 agentic 任務可能成立，但簡單任務用 Haiku 仍更划算
