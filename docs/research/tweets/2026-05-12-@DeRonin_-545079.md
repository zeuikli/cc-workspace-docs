### The Fundamental Insight (You're Not Paying for Tokens, You're Paying for Context)

Every "reduce your AI bill" article online tells you to swap models

That's the WRONG fix

The actual fix is upstream: stop sending tokens you didn't need to send

A typical vibe coder session looks like this:

1. Open Cursor
2. Auto-context loads 47,000 tokens of repo files
3. Ask Claude to "fix the bug in this function"
4. Claude reasons over 47,000 tokens just to find the 30 lines that mattered
5. Claude returns a 200-token fix
6. Cycle repeats 50 times that day

Cost: ~$0.70 per turn × 50 turns = $35/day on a "small" workday

Actual signal: 30 lines that mattered

You didn't pay Claude to fix the bug. You paid Claude to read the entire repo 50 times so it could find 30 lines

Context discipline is the lever. Model selection is downstream of it

### Token Economics 101

There are 4 token categories on every modern AI bill:

- **Input tokens**: everything you send TO the model. Priced per million ($/M input)
- **Output tokens**: everything the model sends BACK. Usually 3-5x more expensive per token than input
- **Cached tokens**: input tokens marked for caching. Priced ~10% of regular input cost
- **Reasoning tokens**: internal "thinking" tokens. You're billed for them even though you don't see them

Approximate pricing as of mid-2026:

- Claude Opus 4.6: ~$15 / $75 per million (input / output)
- GPT-5: ~$10 / $40
- Claude Sonnet 4.6: ~$3 / $15
- Claude Haiku 4.5: ~$1 / $5
- Kimi 2.6 (Moonshot): ~$0.50 / $2

The gap between most expensive and cheapest is roughly 30x on input, 35x on output

### The 5 Token Traps Every Vibe Coder Falls Into

**Trap 1: Re-Sending Your Entire Repo on Every Turn**

A 50-file context = ~80,000 input tokens. At Opus pricing, that's $1.20 per turn. 50 turns/day = $60/day = $1,800/month JUST on re-sending unchanged context

Fix: Turn off auto-context for stable files. Use grep/ripgrep BEFORE asking the model. In Claude Code: rely on the agent's own grep tool instead of front-loading files

Savings: 60-80% on input tokens for stable sessions

**Trap 2: Tool Call Loops That Spiral**

Every "let me check that" from the agent is paying full input cost again. By the time the agent has the answer, you've paid for the same 50,000-token context 5 times

Fix: Batch related tool calls. Ask the agent to plan its tool calls upfront before executing. Summarise tool outputs aggressively. Profile your tool calls for one week

Savings: 3-5x cost reduction on agentic flows

**Trap 3: Running Premium Models on Tasks Cheap Models Could Handle**

You ask Opus to "fix this typo." Model burns 8,000 tokens of reasoning. Cost: $0.60 for a task Haiku would've nailed for $0.02

My agentic refactor loop: Opus end-to-end = $18-24/run. Kept Opus only for planning step, routed 25-30 iteration steps to Kimi 2.6. New cost: $1.40/run. Same workflow, same shipped code, same passing tests

Savings: 95% on cleanup/format/lint tier. 10-15x on long agentic loops

**Trap 4: Streaming When Batched Would Do**

Streaming responses can defeat prompt caching for stable-prefix workflows

Fix: Use BATCHED responses for stable-prefix workflows. Use STREAMING when you want UX feel for interactive coding

Savings: 30-50% on cached-prefix calls when batched correctly

**Trap 5: Context Bloat From "Just in Case" Includes**

Fix: Grep/ripgrep first. If grep doesn't find a reference, the model doesn't need the file. Ask the agent to request files it needs. Don't volunteer them

Savings: 70%+ on input tokens

### The Router Architecture

Routing decision tree:

1. Planning / architecture task? → Premium tier (Opus 4.6 or GPT-5). The 10% of decisions that compound
2. Implementation, code review, refactoring, debugging, serious coding? → Kimi 2.6. Daily driver. 6x cheaper than Sonnet, same quality
3. Long agentic loop with many iterations? → Kimi 2.6
4. Lint, format, single-line edits, trivial fixes? → Utility tier (Haiku 4.5)
5. Boilerplate, autocomplete, stub generation? → Local tier (Qwen 3 via Ollama). Free

### Model Tiers

**Premium Tier**: Opus 4.6 (~$15/$75/M) + GPT-5 (~$10/$40/M) — for architecture decisions, security-critical review, complex refactors, debugging concurrency. About 10% of work

**Utility Tier**: Haiku 4.5 (~$1/$5/M) — lint, format, single-line edits, simple stubs

**Local Tier**: Qwen 3 / Llama 3 via Ollama — $0/token. Autocomplete, boilerplate, syntax fixes only

### 7 Practical Techniques

1. **Enable Prompt Caching everywhere**: cached tokens cost ~10% of regular input. In Claude Code: automatic. In Cursor: settings → models → "use prompt caching". In Aider: `--cache-prompts`
2. **Grep Before Fetching**: grep for symbol/pattern first. 90% of the time, 30 lines is enough
3. **Profile Tool Calls**: log every tool call's input/output token count for one week. Most people cut 30-50% just fixing top 3 worst tool loops
4. **Graduated Skill Pattern**: write SKILL.md files for repeating workflows. "deploy to staging" went from $4/run (Opus) to $0.18/run (Kimi 2.6 + SKILL.md)
5. **Local Models for Boilerplate**: Qwen 3 / Llama 3 via Ollama for autocomplete. $0/token
6. **Summarise Aggressively in Long Sessions**: after every 10-15 turns, ask agent to summarise. 200k-token session compresses to 5k-token summary
7. **Batch Small Requests**: 10 questions one at a time = 10 separate prefix charges. Batch into one prompt: 70-90% savings

**Week 2**: Switch default to Kimi 2.6 — the structural week. Route lint/format to Haiku. Reserve Opus/GPT-5 for planning only → Expected additional savings: 40-55%

Cumulative: 70-85% bill reduction in 30 days

### When To Spend More

Always use Opus / GPT-5 for:
- System architecture decisions
- Security-critical code review

- Debugging concurrency / race conditions

Rule: If the cost of a wrong answer is more than 100x the model cost difference, use the premium model

1. **Context 是主要成本，非模型定價**：80% 浪費來自重複發送未改變檔案（50 files auto-context = $1.20/turn）；model selection 只是 context discipline 的下游問題
