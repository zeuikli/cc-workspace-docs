## 全文

Claude Opus 4.7 is here — and this isn't just an upgrade, it's a shift in how you work with AI.

For the first time, a model doesn't just respond to prompts — it actually respects them.

If you've used Opus 4.6, you've seen the gaps: skipped steps, reordered logic, "smart guesses" you didn't ask for.

Opus 4.7 removes that layer. It follows instructions literally — which means your output quality is now directly tied to how well you prompt.

**核心執行能力提升：**

- 多步驟工作流不漂移
- 輸出前自我驗證
- 長 session 保持一致性

**視覺能力升級：**

支援高解析度輸入（~2,576px）：分析 UI 截圖、提取複雜圖表數據、pixel-perfect 前端協助

**效能：**

在 finance、legal、coding tasks 等真實場景表現更強，產生更乾淨的分析、更好結構的輸出

**新增 xHigh effort mode：**

- xHigh = 實用預設，適合 coding/research/agent workflows
- Higher effort = 更好推理，但也 = 更多 latency + 更多 tokens

**定價：**

- $5 / 1M input tokens
- $25 / 1M output tokens
- 注意：Opus 4.7 使用新 tokenizer，同任務可消耗約 1.0–1.35× 更多 tokens

**新開發者功能：**

- `/ultrareview` → 完整 code review sessions（非只是 linting）
- Auto mode → 長 agent workflow 減少中斷
- Task budgets → 控制 token 消耗

**安全性：**

新 safeguards 偵測並阻擋高風險 cybersecurity 濫用，同時允許合法研究人員透過驗證程序進入。

## 研究摘要

**TL;DR**：Opus 4.7 不是「更聰明」而是「更可靠」，核心升級是字面遵循指令、xHigh effort mode、新 tokenizer 導致 1.0-1.35x token 消耗、和三個新開發者功能（/ultrareview、Auto mode、Task budgets）。

### 核心論點

- Opus 4.6 的痛點：跳步、重排邏輯、不請自來的「聰明猜測」；4.7 的改進是字面遵循指令
- xHigh effort mode 是 coding/research/agent 的實用預設，不是非必要的最高設定
- Tokenizer 更新是被低估的細節：同任務最多多消耗 35% tokens，效率依賴 prompt 品質
- `/ultrareview`、Auto mode、Task budgets 是三個對 cc-workspace 直接相關的新功能

### 關鍵數據 / 工具

- 價格：$5/1M input、$25/1M output（與 4.6 相同）
- Token 消耗：新 tokenizer，同任務 1.0–1.35× 更多 tokens
- 視覺：~2,576px 高解析度支援
- xHigh effort mode（新增，介於 high 和 max 之間）

### 批判性觀察

- 文章對 Opus 4.7 的描述與 workspace 既有的 `.claude/rules/opus47-best-practices.md` 高度吻合，可作為交叉驗證
- 「字面遵循指令」的說法需測試驗證，可能因任務類型而差異明顯
- 1.0-1.35× token 消耗增加是重要的 FinOps 考量，CLAUDE.md 的 token efficiency 規則需要更新
