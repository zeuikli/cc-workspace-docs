## 核心摘要

> Boris Cherny（Claude Code 創始人）在倫敦首次「Code with Claude」活動上的開場演講，宣布 Claude 現在可以在無人值守的情況下自我提示（self-prompting via routines），並展示 AI 開發的指數成長路徑。

### 主要論點

**1. 程式設計回到了「計算機時代」的感覺**
- Boris 以 TI-83 計算機和 eBay 賣寶可夢卡牌為引子，描述程式設計原本的「魔法感」
- 中間年代：compilers、type checkers、build systems 讓「有想法」到「能執行」的距離越拉越長
- 現在：這個距離正在崩縮 — 描述問題，程式就出現了

**2. 真實客戶案例**
- **Spotify**（Nicholas Gustafson 帶領）：background agent 每月合併 1,000+ PRs，遷移時間縮短 90%
- **Vinti**（Felicia Corkuru 創辦）：用 Claude API 讓社工從文書作業解放，兒童寄養家庭認證流程縮短 20 天

**3. 指數 vs 線性採用的落差**
- 模型能力：指數成長
- 企業採用：線性成長
- → 開發者的角色：縮短這個落差，把模型能力轉為實際可用的產品

**4. 里程碑時間軸**
- 2年前：Claude 能寫出不錯的 git commit
- 1年前：首屆 Code with Claude（SF），Opus 4，無人值守跑幾分鐘完成整個 feature
- 6個月前：agent 可以整夜跑完端對端任務
- 上個月：Mythos 讀完整個 OpenBSD source tree，找到一個存活 27 年的漏洞

**5. 今日議程預告**
- **Lisa**：模型層 + 前沿模型動態
- **Angel & Caitlin**：雲端平台 managed agents + 安全部署
- **Boris & Kat**：Claude Code 新功能 — agent view + **routines**（讓 Claude 在你離開電腦時自我提示）

### 關鍵句

> "It's the calculator feeling — except the calculator can write a distributed system."

> "Claude can prompt itself, even when you're away from your computer." ← 這就是 routines / self-prompting 功能
