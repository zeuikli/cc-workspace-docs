## 全文

Tony Stark did not build JARVIS in a day. But he also did not have Claude Code.

#### What JARVIS Actually Is

四個核心功能：
1. Zero-friction 捕捉所有想法
2. 自動連接 vault 中的想法（找出手動找不到的關聯）
3. 從連結產生 content briefs（不再從空白開始）
4. 用你的聲音從 briefs 撰寫內容

#### What You Need Before You Start

```
npm install -g @anthropic/claude-code
export ANTHROPIC_API_KEY=your_key_here
```

無需付費 plugins，只需 Anthropic API 用量費用。

#### Step One: Build the Vault Architecture

```
JARVIS/
├── 00-INBOX/           # 零組織速度捕捉
├── 01-CAPTURES/
│   ├── observations/
│   ├── reactions/
│   ├── patterns/
│   ├── questions/
│   └── numbers/
├── 02-CONNECTIONS/     # 綜合洞見（從多個 captures 生成）
├── 03-BRIEFS/          # 準備好撰寫的 content queue
├── 04-PUBLISHED/       # 附效果數據的存檔
└── 05-CLAUDE/
    ├── CLAUDE.md
    ├── skills/
    └── context/
```

關鍵設計決策：**按 note type 而非 topic 組織**。按 topic 組織時「AI content strategy」和「注意力心理學」永遠不會相遇；按 type 組織則都會進入 patterns/ 讓 Claude 自動找到連結。

#### Step Two: Install the Essential Plugins

- Templater — 動態模板（含邏輯）
- Dataview — 像資料庫一樣查詢 vault
- QuickAdd — 一個快捷鍵零摩擦捕捉
- Obsidian Git — 每小時自動備份到 GitHub

#### Step Three: Build the CLAUDE.md File

CLAUDE.md 是整個系統最重要的檔案，格式包含：

```markdown
# JARVIS SYSTEM — CLAUDE.md

## Identity
My name is [YOUR NAME]. I am [WHAT YOU DO].
My audience is [WHO FOLLOWS YOU].

## Vault Structure
- 00-INBOX: unprocessed captures
- 01-CAPTURES/observations: things I noticed
- 01-CAPTURES/patterns: the same principle appearing in two different domains
- 02-CONNECTIONS: synthesized insights
- 03-BRIEFS: content ready to write
...

## My Voice
[具體描述你的寫作風格]
Example: Short punchy sentences. Hard stops. No AI-sounding language.

## Hard Rules
- Never read/modify .env files
- Never modify 04-PUBLISHED without explicit instruction
- Never use [WORDS YOU HATE]

## Your Primary Jobs
1. Process INBOX — sharpen raw captures, file correctly
2. Run connection sessions — find non-obvious relationships
3. Generate briefs — turn connections into content briefs
4. Write content — produce finished posts in exact voice
5. Log performance — update PUBLISHED notes with data
```

#### Step Four: Connect Claude Code to Your Vault

```bash
cd ~/path/to/JARVIS-vault
claude
```

Claude Code 自動找到 CLAUDE.md，載入 permanent context。

#### Steps Five-Eight: Core Skills, Daily Ritual, Weekly Sessions, Performance Loop

- Step 5: Skills for INBOX processing、connection finding、brief generation、voice matching
- Step 6: 每日 JARVIS Ritual（每天 10 分鐘）
- Step 7: 每週 connection session
- Step 8: 用效果數據關閉 feedback loop，讓 JARVIS 學習什麼有效

## 研究摘要

**TL;DR**：以 Obsidian + Claude Code 建構個人知識系統「JARVIS」的完整逐步指南，核心設計決策是「按 note type 而非 topic 組織」，讓 Claude 自動找出跨領域連結。

### 核心論點

- 最重要的架構決策：按 note type（observations/patterns/questions/numbers）而非 topic 組織，讓 AI 跨領域連結成為可能
- CLAUDE.md 是系統核心：詳細描述身份、vault 結構、寫作風格、Hard Rules，讓 Claude 像「管理了數月的系統」而非「陌生資料夾」
- 三階段工作流：捕捉（INBOX）→ 連結（CONNECTIONS）→ 生產（BRIEFS → PUBLISHED）
- Performance loop 讓系統持續學習：PUBLISHED 資料夾存效果數據，形成 feedback

### 關鍵數據 / 工具

- Obsidian（免費個人使用）+ 4 個 plugins（Templater、Dataview、QuickAdd、Obsidian Git）
- 每小時 Git 自動備份
- 每日 10 分鐘 ritual
- `npm install -g @anthropic/claude-code` 安裝指令

### 批判性觀察

- 文章風格偏行銷（「Tony Stark」開頭），但技術細節確實完整
- CLAUDE.md 模板的 Voice 部分對工程師來說過於文字工作者取向，需客製化
- 對本 workspace 的直接應用有限（cc-workspace 主要用於 SRE/DevOps），但 CLAUDE.md 設計原則和 Hard Rules 概念值得借鑑
