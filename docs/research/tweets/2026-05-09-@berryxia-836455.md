## 全文

YC CEO Garry的这套知识复利效应就像滚雪球一样！
系统开源免费，逻辑清晰。
强烈按头学习了！

Garry Tan（Y Combinator CEO）深夜 coding 到 2AM 的真实原因来了！

AI 让他重新变回 builder，亲手造了一个 复利效应 的个人 AI 操作系统——GBrain。

他不再把 AI 当聊天窗口，而是当成了真正的"第二大脑 / 神经系统"。

过去 5 个月，他用这个系统完成了 20+ 本书的深度镜像、每次会议的自动预习、实体自动传播更新，以及 100,000 页结构化知识的持续生长。

核心案例拆解：

1. Book Mirror（书读我）：拿 Pema Chödrön 的《When Things Fall Apart》为例，系统把 22 章全部提取出来，每章同时生成作者观点 + 针对 Garry 个人生活（移民家庭背景、YC 工作、治疗师对话、创始人对话）的精准映射。

整个过程 40 分钟完成，输出 3 万字双栏对照。后面每本书都会自动调用之前的镜像，上下文越来越厚。

2. Meeting Prep（会议自预习）：Demis Hassabis 来 YC 聊天前，2 分钟内系统就拉出 Demis 完整脑页、AGI 时间线观点、最新传记要点、与 Garry 观点的重叠/分歧，以及 3 个演示脚本和对话钩子。

3. Skillify 元技能：遇到重复工作流就说"技能化它"，系统自动把一次手动操作提炼成可复用 skill（含触发器、边缘案例、测试），注册到 resolver。

Book Mirror、Meeting Ingestion、Enrich 等 100+ skills 就是这样迭代出来的。

架构极简清晰：
- Thin Harness：OpenClaw / Hermes Agent，只负责路由，几千行代码。
- Fat Skills：每个 skill 是一个 markdown 文件，专注单一任务，可组合、可迭代。
- Fat Data：100,000 页结构化脑页（人、公司、会议、书、文章全部链接），每天 100+ cron 自动更新。
- 模型可换：Opus 4.7 1M 做 precision，GPT-5.5 做 recall，DeepSeek V4-Pro 做创意，按 skill 自动选择。

Garry 直接把整套栈开源：GBrain + GStack + OpenClaw / Hermes Agent，GitHub 上 87k+ stars 的 coding 框架也包含在内。

他核心观点是：未来属于那些把 AI 建成 复利系统的人，不属于只用大厂聊天工具的人。
区别就像日记 vs. 真正的神经系统。

对所有想把 AI 真正融入工作流、做个人知识管理、或者构建 Agent 系统的 builder 来说，这篇长文 + 开源项目干货拉满，值得立刻去读和 fork。

## 研究摘要

**TL;DR**：Garry Tan 以 Thin Harness + Fat Skills + Fat Data 三層架構打造 GBrain，實現 100k 頁知識複利增長。

### 核心論點

1. **Thin Harness 原則**：Harness 只負責路由（幾千行），複雜邏輯全部下沉到 Skill 層，避免 Harness 膨脹成維護負擔。
2. **Fat Skills**：每個 skill 是獨立的 markdown 檔案，含觸發器、邊緣案例、測試規格；100+ skills 透過「Skillify」元技能迭代產生。
3. **Fat Data 複利效應**：100k 頁結構化知識庫每日 cron 自動更新，書、人、會議、文章全部交叉鏈接，新輸入自動調用歷史 context。
4. **模型選型依 skill 切換**：Precision → Opus 4.7 1M；Recall → GPT-5.5；Creative → DeepSeek V4-Pro。不綁定單一模型。
5. **知識作為複利資產**：每本書鏡像（Book Mirror）後自動與個人脈絡綁定，Meeting Prep 2 分鐘內提取 + 交叉對比觀點。

### 關鍵數據 / 工具

- GBrain + GStack + OpenClaw / Hermes Agent（開源）
- GitHub 87k+ stars coding 框架
- 100,000 頁結構化知識庫；100+ cron 每日更新
- Book Mirror：22 章 / 40 分鐘 / 3 萬字雙欄輸出
- Meeting Prep：2 分鐘完成人物檔案 + 觀點差異 + 3 個對話鉤
- 模型組合：Opus 4.7 1M / GPT-5.5 / DeepSeek V4-Pro

### 批判性觀察

1. **可行動性有落差**：Garry 的 GBrain 建立在 5 個月累積 + 100k 頁既有資料上，複製「書讀我」類功能需要先建立個人化數據層；cc-workspace 目前的 `research/` 目錄體量（數十篇）遠低於觸發複利效應的臨界點。
2. **Thin Harness 論點高度吻合 cc-workspace 現有設計**：cc-workspace 本身即採用 Thin Harness（`settings.json` 路由）+ Fat Skills（`.claude/skills/`）架構，GBrain 架構可視為當前設計方向的外部驗證。
3. **「開源」細節未驗證**：推文提到 GBrain 開源但未附連結；berryxia 為二手轉述，原始 Garry Tan 文章連結也未出現在推文中，需查原始來源才能確認技術細節可靠度。
