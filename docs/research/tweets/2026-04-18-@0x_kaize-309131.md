## 研究摘要

**TL;DR**：整理 10 個 Claude Code 生態系 GitHub repo 精選清單，涵蓋 skills、plugins、subagents 與 orchestration 框架。

### 核心論點

1. **awesome-claude-code**（hesreallyhim）是生態系主索引，28.5k stars，僅允許 Claude 本身提交 PR，涵蓋 skills / hooks / commands / orchestrators / plugins 分類。
2. **ComposioHQ 雙 repo**（awesome-claude-skills + awesome-claude-plugins）提供 50+ 生產就緒 skills 與 plugin 擴充，跨 Claude Code / Codex / Cursor / Gemini CLI 可用。
3. **VoltAgent 雙 repo**（awesome-agent-skills + awesome-claude-code-subagents）強調非 AI 生成、真實工程團隊驗證的 skills 與 subagent bundles，1000+ items。
4. **obra/superpowers** 號稱最熱門社群 skills library，148k stars，有結構化工作流（brainstorm → spec → plan → TDD → review → merge）與 subagent 新 context 隔離。
5. **Claude-Flow**（ruvnet）定位企業級多 agent 編排平台，11.4k stars，含持久記憶。

### 關鍵數據 / 工具

- awesome-claude-code：28.5k stars
- awesome-claude-prompts：4.2k stars，413 forks
- obra/superpowers：148k stars（異常高，需驗證）
- Claude-Flow：11.4k stars
- awesome-agent-skills：1000+ skills
- 所有 repo 均聲稱跨工具相容（Claude Code / Codex / Cursor / Gemini CLI）

### 批判性觀察

1. **148k stars 存疑**：obra/superpowers 聲稱 148k stars 遠超 awesome-claude-code 的 28.5k，但這個數量在 GitHub 極為罕見（僅有 freeCodeCamp / VS Code 等少數 repo 達此量級），高度可疑，需實際驗證。
2. **內容品質差異大**：部分 repo（VoltAgent 強調「非 AI 生成」）暗示其他清單存在 AI 灌水問題；cc-workspace 整合前應先評估各 skills 的實際品質，而非依星數決策。
3. **對 cc-workspace 的應用**：awesome-claude-code 和 ComposioHQ skills repo 最值得探索，可作為 `.claude/skills/` 新 skill 的靈感來源；但本 workspace 已有嚴格的 skill-authoring 規範，外部 skill 需通過 `evaluate-import` 審查再整合。
