## 核心摘要

> 約 100 分鐘的學術演講，完整拆解 LLM 訓練流程——從 Pre-training 到 Post-training（SFT + RLHF），強調資料、評估、系統三項往往比架構更關鍵。

### 重點結構

**Pre-training（00:00 – 01:00:00）**
- **語言建模任務**：預測下一個 token（next-token prediction），模型本質是 token 序列的機率分佈
- **Tokenization**：BPE（Byte Pair Encoding）subword 切分；比 word-level 更通用，比 char-level 更高效
- **訓練資料**：Common Crawl（全網爬蟲）+ 品質過濾；學術界關注架構，工業界更看重資料與系統
- **Scaling Law**：增加 compute（模型大小 × token 數）→ 損失持續下降；Chinchilla 法則指出最優比例（token 數應為參數量的 20 倍）
- **開源代表**：LLaMA 系列（當時最強開源模型）

**Post-training（01:00:00 – 01:38:00）**
- **SFT（Supervised Fine-Tuning）**：在人類標注的「問題→理想答案」配對上做語言模型微調；ChatGPT 的關鍵跳躍（GPT-3 → ChatGPT）
- **RLHF（Reinforcement Learning from Human Feedback）**：人類對模型輸出排序 → 訓練 Reward Model → PPO 強化學習；副作用：RLHF 越多，輸出越長（reward hacking）
- **Alignment**：拒絕有害請求（toxic content、有害行為）；預訓練資料量大但非所需，SFT 資料少但精準

**Systems（01:38:00 – 結束）**
- GPU 計算速度提升快於記憶體與通訊頻寬，成為主要瓶頸
- 分散式訓練：資料並行、模型並行、Pipeline 並行三種策略

### 核心洞察（講者強調）
> 學術界過度聚焦架構；工業界真正勝負在**資料品質、評估指標設計、系統效率**三點。
