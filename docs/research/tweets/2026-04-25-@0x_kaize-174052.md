Most people think of Claude Code as "just another AI coding tool."

I thought the same - until I discovered configs, plugins, skills, and subagents.

Before that, I was writing the same prompts over and over. Losing context every session. Wondering why other devs were 5x more productive with the same tool.

So I decided to figure out how to work better. The answer: REPOSITORIES.

I put together a collection of the 100 best repositories:

## 研究摘要

**TL;DR**：100 個 Claude Code 生態系 GitHub repo 精選，分 11 類（Awesome Lists、官方 repo、Skills、Agents、MCP、Orchestration、Memory、Slash Commands、Guides、IDE、特化工作流），附 GitHub 連結與星標數，是現有最完整的 Claude Code 生態系索引。

### 核心論點

- Claude Code 生態系比大多數使用者認知的龐大，configs/plugins/skills/subagents 是少數人才掌握的槓桿層
- 分層分類策略：從 Awesome Lists 作為探索入口 → 官方 repo 作為穩定底座 → 社群 Skills/Agents 擴展能力
- Memory/Context 類別是解決「每 session 從零開始」問題的關鍵，Repomix（20.9k stars）是高度成熟的解決方案
- 規模指標：obra/superpowers 148k+ stars、anthropics/claude-code 55k+ stars、claudewave 追蹤 3800+ repos

### 關鍵數據 / 工具

- obra/superpowers：148k+ stars（最高星標的 Skills 庫）
- anthropics/claude-code：55k+ stars（官方 CLI）
- awesome-claude-code (hesreallyhim)：28.5k+ stars，only Claude can submit PRs
- everything-claude-code：30 agents、136 skills、60 slash commands、1282 tests、98% coverage
- Repomix：20.9k+ stars（codebase → AI-readable file）
- claude-context-mode：315 KB → 5.4 KB（98% context reduction）
- ClaudeWave：即時追蹤 3800+ repos，每 12 小時更新

### 批判性觀察

- 部分 repo 星標數可能誇大（作者未交代篩選標準，star count 容易被刷）
- 100 個 repo 的「精選」清單對一般使用者而言資訊量過載，缺乏優先順序建議（哪 5-10 個最重要）
- 作者提供 Telegram 頻道而非公開驗證，難以交叉確認所有描述的準確性
