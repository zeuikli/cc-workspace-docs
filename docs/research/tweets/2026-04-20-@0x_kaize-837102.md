## 研究摘要

**TL;DR**：10 個精選 Claude Code 生態系 GitHub repo，涵蓋 agent 協作、平行執行、MCP、subagent 集合與 prompt engineering 指南。

### 核心論點

1. **agent-in-agent 模式**：claude-code-mcp 讓 Claude Code 自身變成 MCP server，實現遞迴委派；與本 workspace 的 subagent-strategy 直接對應
2. **Subagent 集合資源**：davepoon/claude-code-subagents-collection（2.2k stars）與 vijaythecoder/awesome-claude-agents（3.7k stars）提供 100+ 及多領域現成 subagent
3. **平行執行框架**：claude_code_agent_farm 提供在多個 Claude Code session 同時跑的 orchestration 框架，系統性提升 codebase
4. **背景記憶 agent**：claude-subconscious（2.4k stars）監視 session、讀取檔案、隨時間累積 memory，解決跨 session 記憶問題
5. **語義搜尋 MCP**：claude-context 提供語義程式碼搜尋，大型 repo 不需手動 breadcrumbing 即可找到正確檔案

### 關鍵數據 / 工具

- `vijaythecoder/awesome-claude-agents`：3.7k stars
- `davepoon/claude-code-subagents-collection`：2.2k stars、100+ production-ready subagents
- `0xfurai/claude-subconscious`：2.4k stars
- `ThamJiaHe/claude-prompt-engineering-guide`：涵蓋 100+ curated skills、220+ verified sources
- 系列累計已分享 30 個 repo，第 4 部分即將推出

### 批判性觀察

1. **精選品質良莠不齊**：star 數從 2.2k 到 3.7k 不等，但 obra-superpowers-marketplace、ccpm、claude-code-handbook 無 star 數佐證，實際維護狀況未知，需人工驗證
2. **與本 workspace 高度重疊**：claude-subconscious 的記憶功能、claude_code_agent_farm 的平行執行，本 workspace 已有 Auto Memory + 平行 subagent 策略實現相似功能，整合價值需評估差異點
3. **無技術深度說明**：所有條目僅 3 個 bullet 描述，缺乏實際設定步驟、效能數據或真實案例，主要作為 discovery 清單而非 how-to 指南
