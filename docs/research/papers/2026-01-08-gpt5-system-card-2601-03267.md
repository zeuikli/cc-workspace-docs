## Abstract

GPT-5 is a unified system combining a fast primary model for routine queries and a deeper reasoning model for complex problems, with a real-time router selecting between them. The system demonstrates improvements in hallucination reduction and instruction following, features "safe-completions" safety training, and classifies the reasoning model as High capability in biological and chemical domains under OpenAI's Preparedness Framework.

---

## Architecture

GPT-5 consists of multiple models operating as a unified system:

| Model | Role |
|-------|------|
| gpt-5-main | Fast model for most queries |
| gpt-5-main-mini | Fallback when usage limits reached |
| gpt-5-thinking | Deep reasoning for hard problems |
| gpt-5-thinking-mini | Cost-efficient reasoning |
| gpt-5-thinking-nano | Minimal-cost reasoning |
| gpt-5-thinking-pro | Maximum reasoning effort |

**Real-time router**: Selects model based on conversation type, complexity, tool needs, and explicit intent. Continuously improved through user feedback.

---

## Key Safety Innovation: Safe-Completions Training

Rather than binary refusal approaches, GPT-5 implements **output-centric safety training** that "maximizes helpfulness subject to the safety policy's constraints." This is particularly beneficial in dual-use domains (biology, cybersecurity): instead of refusing, the model completes the request safely.

- Production benchmarks (harder): 0.755–0.958 range

## Preparedness Framework Assessment

### Biological & Chemical Capabilities: **High (precautionary)**

OpenAI designated gpt-5-thinking as High capability despite lacking "definitive evidence" that the model helps novices create severe biological harm:

- Long-form biorisk questions: synthesized information across all five threat creation stages
- Multimodal virology troubleshooting: **exceeded median expert baseline (22.1%)**
- ProtocolQA open-ended: underperformed PhD expert consensus (54%)
- TroubleshootingBench: marginally above OpenAI o3

### Cybersecurity Capabilities: **Does not meet High threshold**

- CTF challenges: comparable to prior models, no significant improvement
- Cyber Range scenarios: unable to solve challenges unaided
- Pattern Labs: 51% evasion success, 35% vulnerability discovery, 49% network simulation

### AI Self-Improvement: **Does not meet High threshold**

- SWE-bench Verified: highest-scoring at time of evaluation
- OpenAI PRs: strongest performance among tested models
- MLE-Bench: participated in evaluation

## Red Teaming Results

**Violent Attack Planning** (25 expert red teamers):
- gpt-5-thinking rated **safer 65.1% of the time vs. OpenAI o3** (large effect size)

**External Assessments**:
- Microsoft AI Red Team: gpt-5-thinking showed "one of the strongest AI safety profiles" across 18 harm categories
- Gray Swan prompt injection benchmark: state-of-the-art performance
- SecureBio: mitigated versions refused all bioweaponization prompts

## Safeguards Implemented

**Multi-layered defense**:
- Model-level training on refusal and safe completions
- System-level protections and account-level enforcement
- API access controls and Trusted Access Program
- Multiple red teaming approaches including government-level assessments

---

## Limitations and Ongoing Challenges

- gpt-5-main regressions in certain content safety categories (hate/threatening, sexual/exploitative)
- Instruction hierarchy vulnerabilities
- Persistent deception risks despite mitigations
- Reasoning model (CoT) transparency remains an open challenge
