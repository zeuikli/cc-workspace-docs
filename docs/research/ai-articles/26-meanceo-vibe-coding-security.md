# Vibe Coding is Great for Validation and Terrible for Unreviewed Production Code

## 核心摘要

Violetta Bonenkamp（Mean CEO 和 CADChain 創辦人）的文章探討了「氛圍編程」在創業公司中的應用與風險。她強調氛圍編程在驗證市場需求時極具價值，但在未經審查的生產環境中卻危險重重。

### 核心立場

作者明確指出，氛圍編程適合用於低風險的驗證工作，但絕不應用於涉及客戶信任、支付、身份驗證或敏感資料的系統。她提出了一個簡潔的原則：「Vibe code the question. Engineer the answer.」（用氛圍編程提出問題，用工程方法解決問題）。

### 安全負債與隱性風險

文章詳細說明了「氛圍編程安全負債」的概念——即創辦人使用 AI 編碼工具快速構建應用程式，卻未能理解、測試或保護程式碼所產生的隱性風險。這些風險包括破碎的存取控制、硬編碼的密鑰、弱身份驗證、缺乏審計日誌等問題。這不只是技術問題，而是對客戶信任和法律責任的直接威脅。

### 實證數據支持

作者援引了多項研究：喬治亞理工大學掃描了超過 43,000 個安全建議，發現 74 起確認案例；Veracode 2025 報告指出 45% 的 AI 生成程式碼樣本未能通過安全測試，並引入了 OWASP Top 10 漏洞。這些統計數字有力證實了安全負債的真實性和規模。

### 適用與不適用場景

**適合使用氛圍編程的場景：**
- 著陸頁面
- 演示流程
- 內部管理工具
- 原型設計
- 銷售計算器
- 測試性工作流程

**不適合使用的場景：**
- 身份驗證
- 授權
- 支付系統
- 客戶記錄
- 健康資訊
- 財務資料
- 法律文件
- 生產資料庫

## 操作檢查清單與責任

作者為非技術創辦人和技術創辦人分別提供了詳細的預發佈檢查清單。

**非技術創辦人應提出的具體問題：**
- 密鑰存儲在哪裡？
- 收集了什麼客戶資料？
- 哪些欄位是私有的？
- 登出後會發生什麼？

**技術創辦人應重點審查的項目：**
- 環境變數管理
- 資料庫規則
- 檔案上傳限制
- 這些「無聊但關鍵」的部分直接影響系統的安全邊界

## 安全設計理念與成熟度

文章強調「Secure by Design」並非僅限於大企業，而是每個創辦人都應採納的實踐原則。這包括了解所構建的內容、知道資料流向、明確存取權限、審查變更流程等基本責任。Bonenkamp 在 CADChain（工程檔案管理平台）的經驗中指出，追蹤變更、限制權限、分離測試和生產資料的重要性。

### 女性創辦人視角

文章指出氛圍編程能幫助歐洲女性創辦人更快地驗證想法，無需等待資本或技術合作者，但這種便利不應演變為對安全責任的逃避。速度對創業至關重要，但未經審查的速度會轉變為延遲的賬單。
