## 繁體中文摘要

### 研究概述

COMPOSITE-STEM 是一個包含 70 個專家撰寫任務的 Benchmark，涵蓋物理（20 題）、化學（23 題）、生物（20 題）、數學（7 題）。任務由博士級研究人員策劃，在可執行的沙箱環境中評估 AI Agent，而非靜態問答格式。

### 評分方法

- **35 題**：完全比對（Exact-match grading）
- **34 題**：LLM-jury 協議（5 個前沿模型投票判斷語義正確性）
- **1 題**：混合評估

### 測試結果

| 模型 | 準確率（Pass@1）|
|------|--------------|
| Claude Opus 4.6 | 21.4% |
| Gemini 3.1 Pro | 18.6% |
| Grok 4.20-beta | 5.7% |
| GPT-5.4 | 4.3% |

**分析**：表現較好的模型使用更多步驟、更頻繁使用工具、並在需要時安裝套件。

### 執行環境

- Python 3.12 + 標準科學套件（numpy、scipy、rdkit 等）
- 每題 1 小時 timeout
- Harbor 框架 + Terminus-2 harness（支援多模態輸入）
- 容器化沙箱

### 數據集

已公開發布，供進一步研究。

## 評分摘要

| 維度 | 分數 | 理由 |
|------|------|------|
| A. Workspace 可行動性 | 4/10 | 這是一個評估工具而非設計模式；對 cc-workspace 的 harness 設計幫助有限 |
| B. 創新性 | 6/10 | LLM-jury 協議和專家策劃的科學任務有一定新穎性；但 benchmark 設計本身不算突破 |
| C. 證據品質 | 7/10 | 4 個模型的量化結果、詳細的評分方法、明確的任務分類 |
| D. 技術深度 | 6/10 | Benchmark 組成和評估方法描述詳細，但執行環境設計說明有限 |
| E. 泛化性 | 5/10 | 高度特化於 STEM 領域；與一般 coding agent 或 harness engineering 的關聯性弱 |
| **加權總分** | **5.45/10** | 4×0.3 + 6×0.2 + 7×0.2 + 6×0.15 + 5×0.15 = 1.2+1.2+1.4+0.9+0.75 |

**整合決策**：不整合（分數 ≤ 6）

### 低分主因

1. **主因（A 維度低，4/10）**：這是一個評估 benchmark，不包含可直接套用到 cc-workspace 的設計模式、工具或流程改善。
2. **泛化性受限（E 維度，5/10）**：STEM 科學任務高度特化，與 cc-workspace 關注的 harness engineering、coding agent、context management 領域關聯性弱。
3. **對比現有資料**：cc-workspace 的重心在 Agent harness 設計和優化；COMPOSITE-STEM 在評估科學推理能力，是不同的研究方向。

### 保留原因

雖不建議整合，但可作為「AI 能力評估方法論」的參考資料：
- LLM-jury 協議（5 個模型投票）是一種可借鑒的評估設計
- 「最強模型僅達 21% 準確率」說明科學推理在目前仍是 Agent 的重要能力邊界
