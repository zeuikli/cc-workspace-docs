# Inside the Scaffold: A Source-Code Taxonomy of Coding Agent Architectures

- **arXiv**: https://arxiv.org/abs/2604.03515
- **收錄日期**: 2026-05-01
- **發表**: 2026-04
- **重要性**: Priority A — 首次系統性分類 coding agent 架構，提出 5 個通用控制原語

---

## 核心論點

首次對 **13 個開源 coding agents** 進行源碼層級的架構分析，提出跨 12 維度、3 層組織的統一分類學。

## 3 層組織架構

1. **控制架構層**: 執行流程的頂層設計
2. **工具/環境介面層**: 與外部系統互動的邊界
3. **資源管理層**: Context、記憶、狀態的管理

## 5 個通用控制原語

這是所有 coding agent 架構的最小可組合基元集合：

| 原語 | 說明 |
|------|------|
| **ReAct** | 推理-行動交錯（Reason + Act） |
| **Generate-test-repair** | 生成 → 測試 → 修復迴圈 |
| **Plan-execute** | 先規劃後執行，分離關注點 |
| **Multi-attempt retry** | 多次重試策略 |
| **Tree search** | 樹狀搜索（蒙地卡洛等） |

**關鍵發現**: **11/13 agents 混合多個原語**，而非單一模式。

## 12 維度變異來源

- 控制流管道（固定 vs 蒙地卡洛樹搜尋）
- 工具數量（0 到 37 個不等）
- Context 壓縮策略（7 種不同方式）

## 對 Harness 設計的意義

不需要重新發明控制流，而是：
1. **識別需要哪些原語組合**
2. **設計原語之間的轉換邏輯**
3. **測試與可觀測性使演進自動化**

## 與本 Workspace 的關聯

- CLAUDE.md 的 multi-step 實作流程 ≈ Plan-execute + Generate-test-repair 組合
- Subagent 委派策略 ≈ Multi-attempt retry + Plan-execute 組合
- /deep-review skill ≈ Generate-test-repair 的 repair 步驟
