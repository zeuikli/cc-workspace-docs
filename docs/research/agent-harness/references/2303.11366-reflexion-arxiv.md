# Reflexion: Language Agents with Verbal Reinforcement Learning

- **arXiv**: https://arxiv.org/abs/2303.11366
- **發表**: NeurIPS 2023
- **作者**: Noah Shinn, Federico Cassano et al.
- **收錄日期**: 2026-05-01
- **重要性**: Priority A — ReAct 最重要的直接延伸；引入「言語強化學習」不需要更新模型權重

---

## 核心創新：突破 ReAct 的限制

ReAct 的問題：每次執行是獨立的，無法從過去失敗中學習。

Reflexion 解法：引入「言語反思迴圈」——任務完成後，agent 生成文字化的反思，下次嘗試時自動注入這些反思作為 context。

```
嘗試 1 → 失敗 → 反思（"我錯在了 X，下次應該 Y"）
  ↓（反思存入 long-term memory）
嘗試 2 → 帶著反思重試 → 改進
```

---

## 技術架構

### 三層結構
1. **Actor**: 執行任務的 LLM（使用 ReAct 或 Chain-of-Thought）
2. **Evaluator**: 評估 Actor 輸出的模型（外部回饋 or 內部 LLM 評判）
3. **Self-Reflection**: 生成言語化反思並存入 episodic memory

### 記憶組件
- **Short-term memory**: 當前 sliding context window（軌跡 + 反思）
- **Long-term memory**: 外部向量存儲（可選）
- **Episodic memory**: 保存過去試驗的具體反思

## 對 Harness Engineering 的影響

### 洞見 #1：反思機制是 long-running agent 的必要條件
移除反思組件的後果（Generative Agents 實驗）：
- 48 小時內從多日規劃退化為重複的無狀態回應
- 證明記憶 + 反思是維持 agent 行為品質的核心機制

### 洞見 #2：Evaluator 必須與 Actor 分離
Actor 評估自己的輸出會產生「自我稱讚偏差」（self-congratulatory bias）。Reflexion 的三層架構正式建立了這個分離原則，後來被 AHE 的「決策可觀測性」和 Anthropic 的 Planner-Generator-Evaluator 架構繼承。

### 洞見 #3：失敗可以被「語言化」為知識
不需要梯度更新，只需把失敗的原因用文字描述清楚，下一次就能改進。這是 Ratchet 原則（每個失敗沉澱成永久規則）的學術驗證。
