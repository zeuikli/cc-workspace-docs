# Agentic Harness Engineering: Observability-Driven Automatic Evolution of Coding-Agent Harnesses

- **arXiv**: https://arxiv.org/abs/2604.25850
- **收錄日期**: 2026-05-01
- **發表**: 2026-04
- **重要性**: Priority A — 本領域最新 SOTA，直接量化 harness 工程的效益

---

## 核心論點

Harness 改進應由**工具、中介軟體和長期記憶**主導，而非系統提示詞。提出 AHE（Agentic Harness Engineering）框架，透過三個可觀測性柱子自動進化 agent harnesses。

## 三大可觀測性柱子

1. **元件可觀測性（Component Observability）**: 檔案級表示、明確動作空間、可逆性設計
2. **經驗可觀測性（Experience Observability）**: 執行軌跡資料 → 分層證據語料庫
3. **決策可觀測性（Decision Observability）**: 編輯 + 自我宣告預測 + 事後驗證

## 關鍵成果

| Benchmark | 優化前 | 優化後 | 提升 |
|-----------|--------|--------|------|
| Terminal-Bench 2 pass@1 | 69.7% | 77.0% | +7.3 pp（10 次迭代） |
| SWE-bench-verified | baseline | SOTA | 用 12% 更少 token |
| 跨模型族遷移 | — | — | +5.1 到 +10.1 pp |

## 核心洞見

- 將每次編輯轉成可驗證的合約，harness 演進可自動進行
- 超越人工設計的 Codex-CLI（71.9%），達到 77.0%
- 證明 harness 優化的效果與跨模型族遷移能力
