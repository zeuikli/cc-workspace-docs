# Meta-Harness: End-to-End Optimization of Model Harnesses

## Abstract

The paper introduces Meta-Harness, a system that automatically searches for improved code harnesses—the execution frameworks that determine what information language models access and process. Rather than relying on manual engineering, Meta-Harness employs an agentic proposer with filesystem access to prior candidates' source code, execution traces, and performance scores.

Key findings include: on text classification, discovered harnesses outperform state-of-the-art manual designs by 7.7 points while consuming 4× fewer context tokens; on mathematical reasoning, a single harness improves IMO-level problem accuracy by 4.7 points across five models; on TerminalBench-2 coding tasks, automated harnesses rank #1 among Claude Haiku 4.5 agents.

---

## Core Contribution

Meta-Harness addresses a fundamental limitation in existing text optimization approaches: they compress feedback too aggressively through scalar scores, brief summaries, or restricted templates. This paper argues that "useful context should often be accessed adaptively" rather than packed into single prompts. The system provides "full access to execution traces" enabling proposers to form causal hypotheses about failures and test targeted fixes.

---

## Technical Approach

**Search Loop:** The system iteratively:
1. Proposer inspects filesystem containing prior harness code, scores, and traces
2. Generates new harness candidates
3. Evaluates and logs results with full diagnostic information
4. Repeats for fixed iterations

**Key Design Choice:** Rather than summarizing history, the filesystem stores complete artifacts. The proposer queries these via standard tools (grep, cat), reading a median of 82 files per iteration and referencing over 20 prior candidates per step.

**Scale:** Individual evaluations generate up to 10 million tokens of diagnostic data—roughly "three orders of magnitude beyond" prior text optimization work.

---

## Experimental Results

### Text Classification (4 Datasets)
- **Improvement over ACE baseline:** 7.7 percentage points
- **Context reduction:** 4× fewer tokens (11.4K vs 50.8K)
- **Speed advantage:** Matches competing optimizers' final performance in 0.1× evaluations
- **Generalization:** Outperforms hand-designed systems on 9 held-out OOD datasets (73.1% avg accuracy)

### Mathematical Reasoning (200 IMO-Level Problems)
- **Average gain over no retrieval:** 4.7 points
- **Model transfer:** Single harness improves performance across GPT-5.4-nano, GPT-5.4-mini, Gemini-3.1-Flash-Lite, Gemini-3-Flash, and GPT-OSS-20B
- **Approach:** Discovered routing logic directing problems to subject-specific retrieval policies

### TerminalBench-2 Agentic Coding
- **Opus 4.6 performance:** 76.4% pass rate (surpasses Terminus-KIRA at 74.7%)
- **Haiku 4.5 ranking:** #1 among reported agents (37.6% vs prior best 35.5%)
- **Main discovery:** Environment bootstrapping reducing exploratory turns by 2-4 steps

---

## Ablation Studies

The paper demonstrates that raw execution trace access is essential. Compared to scores-only and scores-plus-summary approaches:
- Scores only: 34.6 median accuracy
- Scores + summaries: 34.9 median accuracy  
- Full traces: 50.0 median accuracy

This reveals that "summaries do not recover the missing signal" and may actively harm optimization through information compression.

---

## Qualitative Analysis

The TerminalBench-2 search trajectory reveals sophisticated causal reasoning. The proposer explicitly identified confounded interventions across regressions, hypothesized that prompt modifications (not structural bugfixes) caused failures, tested the hypothesis by isolating variables, and pivoted toward purely additive changes after observing repeated patterns.

---

## Technical Implementation

- **Proposer:** Claude Code with Opus-4.6
- **Base models:** Frozen; vary by domain (GPT-OSS models, Claude variants)
- **Harness representation:** Single-file Python programs modifying prompting, retrieval, memory, and orchestration
- **Typical run:** ~60 harnesses evaluated over 20 iterations
- **Wall-clock time:** Several hours producing readable, transferable strategies

---

## Related Work Positioning

Meta-Harness extends three research areas:

1. **Adaptive context access** (RAG, interleaved retrieval, memory-augmented systems) applied to harness engineering rather than end-user reasoning

2. **Executable code search** over program functions and agent designs, but with unrestricted filesystem access rather than predefined scaffolds

3. **Text optimization** (TextGrad, OPRO, GEPA, AlphaEvolve), but operating at orders-of-magnitude larger information scales with selective rather than monolithic access

---

## Discovered Harness Examples

**Text Classification:** Two complementary strategies emerged:
- Draft-verification: Two-call procedure retrieving confirmers and challengers
- Label-primed query: Single call exposing label space with contrastive local boundaries

**Math Reasoning:** Four-route lexical router assigning problems to subject-specific BM25 policies with problem-specific deduplication and reranking

**Coding Tasks:** Environment bootstrapping gathering OS metadata before initial LLM turn, reducing wasteful exploration

---

## Limitations and Future Work

The authors note the study focuses on one strong proposer (Claude Code) and suggest broader analysis across agents remains valuable. They propose co-evolving harnesses and model weights as natural next steps. The TerminalBench-2 harness, while benchmark-specialized, targets fundamental long-horizon task completion capabilities.

---

## Practical Impact

Discovered harnesses demonstrate genuine transferability: text classification harness generalizes to unseen domains; math retrieval harness improves multiple held-out base models; harnesses produce inspectable code enabling straightforward overfitting detection compared to weight-space optimization.
