## 核心論點

傳統 harness「深埋在控制器代碼中」，難以跨系統移植或科學研究。本論文提出用自然語言表達 harness 行為。

## 兩個核心概念

### Natural-Language Agent Harnesses (NLAHs)
- 人類可讀的 harness 規範
- 可跨系統遷移的格式
- 版本控制友善
- 可科學比較與研究

### Intelligent Harness Runtime (IHR)
執行 NLAH 的運行時，通過三個機制：
1. **顯式合約（Explicit Contracts）**: 明確定義 harness 的行為邊界
2. **持久成品（Persistent Artifacts）**: 跨 session 保持 harness 狀態
3. **輕量適配器（Lightweight Adapters）**: 連接 harness 與不同模型/工具

## 解決的核心問題

- 可移植性：harness 可在不同 agent 框架間遷移
- 可對比性：不同 harness 設計可以被科學比較
- 可研究性：harness 工程成為可重複的學術研究對象

## 與本 Workspace 的關聯

本 workspace 的設計已接近 NLAH 精神：
- CLAUDE.md = Natural-Language Harness 規範
- `.claude/settings.json` = IHR 的顯式合約層
- @rules 系統 = 輕量適配器（按需載入）

NLAH 論文提供了**理論依據**說明為何「CLAUDE.md 作為 harness 規範」是正確的設計方向。
