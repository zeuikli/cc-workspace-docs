# Language Agent Tree Search (LATS)

- **arXiv**: https://arxiv.org/abs/2310.04406
- **發表**: ICML 2024
- **收錄日期**: 2026-05-01
- **重要性**: Priority B — 將 ReAct 的線性 loop 升級為樹狀搜索；Tree search 控制原語的學術依據

---

## 核心創新：統一推理、行動、規劃

LATS = Language Agent Tree Search

將蒙地卡洛樹搜索（MCTS）整合進 LLM agent 框架，同時統一三個維度：

| 維度 | ReAct 的處理 | LATS 的處理 |
|------|------------|-----------|
| **Reasoning** | 線性 CoT | 樹狀節點探索 |
| **Acting** | 依序執行 | 平行探索多條 action 路徑 |
| **Planning** | 隱式（prompt 中） | 顯式 value function 引導 |

---

## 架構設計

### 核心組件

1. **LM-powered Value Function**：語言模型評估每個狀態的「前景」（promise），引導樹搜索方向
2. **Self-Reflection**：訪問被剪枝的分支，提取有用的洞見注入後續搜索
3. **External Environment Feedback**：來自工具/環境的反饋作為搜索的獎勵信號

### MCTS 四步驟（在語言空間的應用）

```
1. Selection: 選擇最有前景的節點（UCB 公式 + LM 評估）
2. Expansion: 展開新的 thought/action 組合
3. Evaluation: LM 評估節點品質
4. Backpropagation: 更新父節點的 value 估計
```

## 實驗成果

| Benchmark | LATS（GPT-4） | 基線 |
|-----------|--------------|------|
| HumanEval pass@1 | **92.7%** | ~80%（純 ReAct） |
| WebShop 導航 | **75.9** | ~70（基線） |

## 對 Harness Engineering 的影響

### 洞見 #1：Tree Search 是所有架構的「探索加速器」
Scaffold Taxonomy（2604.03515）列出的 5 個控制原語中，Tree search 是最複雜也最強大的。LATS 提供了它的學術基礎。

### 洞見 #2：Value Function 可以用語言模型實現
不需要額外訓練的 reward model，只需要讓 LLM「評估這個路徑有多好」。這降低了 tree search 的實作門檻，使其在 harness 中可行。

### 洞見 #3：被剪枝的路徑也有價值
LATS 的 Self-Reflection 機制從「失敗的分支」中提取知識，對應 Ratchet 原則：即使是失敗的嘗試也應該沉澱成知識。

---

## 與 AHE（2604.25850）的關係

LATS 的「值函數引導探索」啟發了 AHE 的「決策可觀測性」：
- LATS：每個節點有明確的 value 估計
- AHE：每次 harness 編輯配對「自我宣告的預測」+ 驗證

兩者都把決策過程「顯式化」，使其可追蹤和可改進。
