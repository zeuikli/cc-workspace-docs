# 7 Practical Ways to Reduce Claude Code Token Usage

## 核心摘要

Kanwal Mehreen 揭示了令牌成本的真實來源：**膨脹的上下文，而非長提示**。使用者為整個會話付費，包括前述訊息、檔案、工具輸出和背景指令，而不僅是可見的提示。本文提供了七項實踐技巧，系統性地減少令牌消耗。

核心理念的轉變：從「如何寫更短的提示」轉向「如何架構更精簡的上下文」。此轉變認知對有效優化至關重要。

### 七大實踐技巧

1. **Switch Models by Task Complexity（按複雜性切換模型）**
   - 常規工作用 Sonnet：測試、編輯、重構
   - 複雜分析和架構決策保留 Opus
   - 機械性任務部署 Haiku（格式化、清理）
   - 成本差異：Opus 比 Sonnet 貴 5 倍/token
   - **實踐**: 為不同任務類型預先配置不同模型

2. **Optimize CLAUDE.md（優化專案指令檔）**
   - 在持久指令檔而非每次對話中保存專案規則
   - 維持精簡文件，著重查詢資訊而非冗長解釋
   - 避免包含冗長會議紀錄或實現歷史
   - 定期審查並移除過時內容
   - **成本影響**: 減少 30-50% 的重複上下文

3. **Delegate to Subagents（委派給子代理）**
   - 在分離的上下文視窗中隔離冗長操作
   - 只在上下文節省能合理化架構開銷時使用子代理
   - 簡單任務可能不會從子代理受益（啟動成本）
   - **計算**: 如果父代理上下文 > 80K，子代理可節省 20-40%
   - **警告**: 5+ 個子代理時重新評估架構

4. **Target Specific Files（指定具體檔案）**
   - 參考精確檔案和行範圍，而非讓 Claude 廣泛探索
   - 使用 plan mode（Shift+Tab）在執行前審查步驟
   - 消除試誤迭代，直接指定目標
   - **成本影響**: 減少 40-60% 的探索開銷
   - **工具**: 使用 `/context` 診斷哪些檔案重複載入

5. **Use /compact Proactively（主動使用壓縮）**
   - 早期壓縮（狀態「健康」時），不要等待警告
   - 移除累積的雜訊（探索、死胡同、失敗嘗試）
   - 保留關鍵決策、假設和代碼片段
   - 維持較清晰的摘要供後續步驟使用
   - **時機**: 每 50-100K tokens 或任務階段變更時壓縮
   - **效果**: 通常減少 60-80% 的令牌

6. **Inspect Context with /context（用 /context 檢查）**
   - 優化前先診斷實際消耗來源
   - 識別「沉默的成本殺手」（重複載入的檔案）
   - 基於測量而非假設進行改進
   - **診斷步驟**：
     - 運行 `/context` 查看頂部消費者
     - 找出哪些檔案在每次轉換時重複加載
     - 考慮使用 skill 或指令檔替代方案

7. **Simplify Tooling（簡化工具堆棧）**
   - 避免載入不必要的集成
   - 僅連接解決重複問題的工具
   - 減少架構開銷
   - **評估**: 每個工具每會話節省 5+ 分鐘？保留；否則移除

## 關鍵實踐

### 令牌優化決策樹

**第一步**：診斷（運行 `/context`）
- 識別頂部 3 個上下文消費者
- 檢查是否有重複加載

**第二步**：快速勝利（實施技巧 2, 4, 6）
- 精簡 CLAUDE.md
- 指定具體檔案範圍
- 停止載入未使用的工具
- **預期**: 20-30% 即時節省

**第三步**：架構優化（技巧 1, 3, 5）
- 模型選擇（按複雜性）
- 子代理委派（複雜多任務）
- 定期壓縮策略
- **預期**: 額外 30-50% 節省

**第四步**：監控（持續）
- 每 10 次會話檢查一次 `/context`
- 追蹤實際 vs 預期令牌使用
- 調整策略

### 成本計算示例

**未優化的會話**：
- 初始 CLAUDE.md 加載: 1.5K tokens
- 工具集合: 2K tokens
- 探索和迭代: 30K tokens
- 總計: ~33.5K tokens

**使用 7 項技巧後**：
- 精簡 CLAUDE.md: 0.8K tokens (-46%)
- 有選擇性工具: 0.5K tokens (-75%)
- 指定檔案 + 早期壓縮: 12K tokens (-60%)
- 總計: ~13.3K tokens（減少 60%）

## 實踐建議

### 快速檢查清單（10 分鐘）
- [ ] 運行 `/context` 確認頂部消費者
- [ ] 檢查 CLAUDE.md 是否包含無關內容

### 中期優化（1 週期）
- [ ] 為常見任務類型建立模型切換規則
- [ ] 建立檔案參考指南而非廣泛探索
- [ ] 設定 50-100K token 檢查點以定期壓縮
- [ ] 建立子代理決策矩陣

### 長期策略（持續）
- [ ] 每月審查 token 使用趨勢
- [ ] 在專案層級應用集體學習（更新 CLAUDE.md）
- [ ] 評估新工具的邊際成本效益
