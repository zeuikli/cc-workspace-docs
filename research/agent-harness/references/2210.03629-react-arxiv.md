# ReAct: Synergizing Reasoning and Acting in Language Models

- **arXiv**: https://arxiv.org/abs/2210.03629
- **發表**: 2022-10（ICLR 2023）
- **收錄日期**: 2026-05-01
- **重要性**: Priority B — Agent harness 的理論基礎論文，所有 agentic loop 的根源

---

## 核心概念

定義 **Thought-Action-Observation** 的交錯迭代模式（ReAct = Reason + Act）。

| 組成 | 說明 | 在 harness 中的對應 |
|------|------|------------------|
| **Reasoning traces** | 內部思考，不影響外部狀態 | LLM 的推理步驟、planning |
| **Actions** | 與外部環境互動 | 工具呼叫、API 請求 |
| **Observations** | 行動的結果 | 工具回傳值、環境反饋 |

## 關鍵洞見

- 推理幫助**歸納、追蹤、更新**行動計畫
- 行動帶來的 Observations 更新推理的前提
- 推理不影響外部狀態（純思考）；行動產生副作用

## 實驗成果

| Benchmark | ReAct 提升 | 對比基線 |
|-----------|-----------|---------|
| HotpotQA | 超越 CoT | 克服幻覺與誤差傳播 |
| ALFWorld | +34% 絕對成功率 | vs RL 方法 |
| WebShop | +10% 絕對成功率 | vs 其他 LLM 方法 |

## 對 Harness Engineering 的意義

1. **Agentic Loop 的理論基礎**: 所有現代 coding agent 都是 ReAct 的變體或組合
2. **Observation 是 Harness 的責任**: 格式化工具輸出、過濾噪聲、注入 context
3. **Action 設計是工具設計**: Tool schema 的精確度決定 Action 的成功率

## 歷史地位

ReAct 是從「LLM 作為問答機器」到「LLM 作為自主 agent」的轉折點。所有後續的 harness 設計（Plan-execute、Generate-test-repair、Tree search 等）都是 ReAct 的延伸與特化。
