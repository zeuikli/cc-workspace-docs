# A-MEM: Agentic Memory for LLM Agents

- **arXiv**: https://arxiv.org/abs/2502.12110
- **發表**: NeurIPS 2025
- **GitHub**: https://github.com/agiresearch/a-mem
- **收錄日期**: 2026-05-01
- **重要性**: Priority B — MemGPT 的後繼；引入 Zettelkasten 動態連結機制，解決靜態向量檢索的局限

---

## 核心創新：Zettelkasten 知識網絡

**Zettelkasten**（德語「索引卡盒」）是德國社會學家 Niklas Luhmann 的筆記方法：每則筆記有唯一 ID，透過雙向連結形成知識網絡，整體大於各部分之和。

A-Mem 將這個原理應用到 agent 記憶：每次存入的記憶不只是孤立的 embedding，而是與其他記憶形成動態連結。

---

## 技術架構

### 記憶生成（Memory Creation）
新記憶加入時，系統自動生成結構化筆記，包含：
- **上下文描述**（contextual description）
- **關鍵詞**（keywords）
- **標籤**（tags）
- **元數據**（metadata: 時間戳、來源、重要性）

### 動態連結（Dynamic Linking）
LLM 判斷新記憶與現有記憶之間是否應建立連結：
- 主題相關性
- 因果關係
- 時間鄰近性
- 概念互補性

### 智能檢索（Intelligent Retrieval）
```
查詢 → 提取 query embedding
  ↓
語義搜索找到相關記憶
  ↓
同一「知識盒」中的連結記憶也自動載入
  ↓
形成豐富的上下文網絡
```

## 與 MemGPT 的比較

| 維度 | MemGPT | A-Mem |
|------|--------|-------|
| 記憶存儲 | 三層（main / recall / archive） | 互連知識網絡 |
| 檢索機制 | 向量搜索（獨立 embedding） | Zettelkasten 動態連結 |
| 記憶關係 | 孤立節點 | 雙向連結圖 |
| 更新策略 | 靜態存儲 | 動態演進（新記憶影響舊連結）|

## 對 Harness Engineering 的影響

### 洞見 #1：記憶不應是「點」而應是「圖」
孤立的記憶片段需要上下文才能發揮價值。A-Mem 的連結機制讓相關知識自動關聯，減少 agent 需要的搜索次數。

### 洞見 #2：記憶創建時的質量決定檢索質量
在記憶存入時花時間生成完整的結構化描述，比存入後優化檢索更有效。

### 洞見 #3：LLM 能夠判斷記憶的連結關係
不需要規則引擎，LLM 本身能判斷哪些記憶應該相互連結，大幅降低記憶系統的工程複雜度。
