## 全文

看完这篇文章让你对 claude code 的理解和使用技巧超过 99% 的人。

就在深夜，claude code 发布最新版本，支持了最新的功能 agent view 以及 /goal 命令。这绝对是 claude 历史上最伟大的更新，它彻底颠覆了人类在编程当中的定位，可以称为另一个"cursor 时刻"！

这已经不是"AI 帮你写代码"那个层面的事了。这是你终于可以像管理一个开发团队一样，管理多个 AI agent 同时干活。

## 先说 Agent View 是什么？

你在终端输入 `claude agents`，就打开了一个"后台任务面板"，所有你扔给 Claude Code 的后台任务，全在这一个屏幕里：
- 哪些正在跑（working）
- 哪些在等你回复（awaiting input）
- 哪些已经完成（completed）

关键是，每个 agent 都在独立的 git worktree 里工作。agent A 在 feature-auth 分支改登录模块，agent B 在 bugfix-api 分支修接口问题，agent C 在 refactor-ui 分支重构页面。它们互不干扰，不会因为改同一个文件打架。

你可以用快捷键快速切换、预览、回复任何一个 agent，不用开 10 个终端窗口，一个界面管理全部。

## 再说 /goal 命令，这个更牛了

以前你让 AI 干活，它干到一半经常停下来问你："接下来我该做什么？"

/goal 命令就是给 agent 设定一个目标，让它自己跑到目标达成为止。比如你在会话里输入：

```
/goal 所有测试通过且 CI 变绿
```

然后 Claude Code 会装一个 session 级的 Stop hook，在目标没达成之前，它不会停。它会自己修代码、跑测试、看报错、再改、再跑，一直到所有测试绿了为止。

这才是真正的"自动化"。不是"帮你写 50% 的代码"，是"你定义目标，它负责跑通全流程"。

## 实战场景：一个人怎么用这套组合拳

假设你现在要做一个重构项目，涉及 3 个模块：认证、API、前端 UI。

```bash
claude --bg -w refactor-auth "重构 src/auth/ 模块，改用新的 token 机制"
claude --bg -w refactor-api "重构 src/api/，统一错误处理"
claude --bg -w refactor-ui "重构 src/components/，应用新设计系统"
```

每个 agent 自动在独立的 worktree 和分支上工作。给每个 agent 设定目标：

```
/goal 所有单元测试通过，代码审查无警告
```

关掉电脑，去健身 / 开会 / 睡觉。agent 会自己跑，遇到问题自己调试，直到目标达成。

回来后一键查看进度：`claude agents`，看到 3 个 agent 都是 completed 状态。

## 快捷键速查（agent view 功能）

进入 Agent View 后：
- ↑ / ↓ 选择会话
- Space peek 预览 + 直接回复（不离开列表）
- Enter 进入完整会话
- Ctrl+T 置顶重要任务
- Ctrl+S 按目录分组
- Ctrl+X 连按两次 删除会话
- ? 显示所有快捷键

把当前会话扔到后台：输入 `/bg`，清除目标：`/goal clear`

## 为什么说这是又一个 cursor 时刻

因为过去 cursor 带给我们用 AI 的心智模型是：AI 是工具，我是使用者。但 Agent View + /goal 把这个关系改成了：AI 是团队成员，我是 PM。

你不用教它每一步怎么做，你只需要：
- 拆解任务（这个本来就是自己的活）
- 分配给不同 agent（每个 agent 有独立上下文和目标）
- 定期 check 进度，必要时介入调整

Anthropic 自己内部现在大部分代码已经是 Claude Code 写的，工程师的工作重心变成了产品思维 + agent 编排。

## 研究摘要

**TL;DR**：Claude Code 新功能 Agent View + /goal 實現並行多 Agent 管理，人類角色從寫碼轉為 PM

### 核心論點

1. **Agent View**：`claude agents` 指令開啟統一後台面板，管理多個並行執行的 agent
2. **git worktree 隔離**：每個 agent 在獨立分支工作，避免文件衝突
3. **/goal 命令**：設定目標後 agent 自主執行直到達成，安裝 session 級 Stop hook
4. **角色轉換**：開發者從執行者 → PM，負責拆解任務、分配、驗收
5. **`--bg -w` 旗標**：背景啟動 + 指定 worktree，一次開多個 agent

### 關鍵數據 / 工具

- 指令：`claude agents`、`claude --bg -w <branch> "<task>"`、`/goal <condition>`、`/bg`、`/goal clear`
- 快捷鍵：Space（預覽回覆）、Ctrl+T（置頂）、Ctrl+S（分組）、Ctrl+X×2（刪除）
- 架構：git worktree 分支隔離 + session Stop hook

### 批判性觀察

1. **過度誇張「最偉大更新」**：功能有實際價值，但 clickbait 語氣降低了可信度；需自行測試驗證
2. **「自主跑到完成」有邊界**：/goal 依賴 Stop hook，若任務無法被測試驗收（如「寫一篇文章」），效果有限
3. **worktree 合並責任未提**：多個 agent 各自完成後的 PR review 和 merge 衝突處理留給使用者，非全自動
