每个小白和达人必看！一次设置，永久生效：Claude Skills 从入门到精通最佳实战经验！ — @berryxia ​

你每天都在做这件事，却从没发现它有多浪费时间 ​

[圖 1] 想象一下这样的早晨。 你打开 Claude，要写一篇公司博客。于是你开始打字：

“我们是一家做 B2B SaaS 的公司，主要服务中小企业，语气要专业但不要太正式，受众是非技术背景的创业者，不要用‘赋能’‘生态’‘闭环’这些词，标题要具体不要抽象，段落不超过四句话……” 说完这些，Claude 终于理解了你是谁，对话才能正式开始。 第二天，你打开一个新对话，要写一封销售邮件。 “我们是一家做 B2B SaaS 的公司……” 你又开始打了。 这不是偶发状况，这是每一个重度使用 Claude 的人的日常。而且你可能根本没意识到，因为每一次解释都只要几分钟，看起来没什么大不了。 但问题来了： 📊 每天打开 5 个新对话 ⏱️ 每次花 3 分钟解释背景 📅 一周 = 105 分钟 🚨 一年 = 将近 90 小时 这 90 小时，全部用在了告诉 Claude “你是谁” 上面。 💡 Claude Skills 是这个问题最彻底的解法 不是优化解，是彻底消灭的解法。 @aiedge_ 运营着三家公司，在 2026 年 3 月的一篇文章里说： “如果这个月我在 X 上发的 200 多条内容里，你只能带走一件事，就带走 Skills。” 他不只是在用 Skills，他在每家公司的每个部门都强制要求大家构建并使用它们。 📖 这篇文章的内容： ✅ Skills 是什么 ✅ 怎么建 ✅ 怎么建好 ✅ Anthropic 在 2026 年 3 月刚推出的三个重要升级

什么是 Claude Skills ​

[圖 2] 一句话说清楚：Claude Skills 是把你的指令集打包成文件，随时调用的复用模块 技术上，它是一个 Markdown 文件，里面写着 Claude 完成某类任务需要知道的一切——你的身份、你的受众、你的标准、你的禁忌、什么叫好输出。你把这个文件上传到 Claude 账户，之后每次只需要一句话触发它：

“用我的品牌音调 Skill，写一篇关于产品发布的 LinkedIn 帖子。” Claude 会自动调取文件里的全部上下文，产出的结果就像你已经把所有背景解释过一遍。但你什么都没有解释。 这背后有个值得仔细想的逻辑。Claude 本身的能力在不断进化，但大多数人的使用方式却停在原地——每次对话从空白开始，把 Claude 当临时工具使用，用完即扔。Skills 改变了这个模式。它把你的专业判断、工作偏好和操作标准固化成可调用的资产，让 Claude 不再是一个陌生的临时工，而是一个已经接受过充分培训的专职助理。 类比：就像一家餐厅的 SOP 手册。新厨师来了，不需要主厨每道菜都在旁边盯着教，手册里有食材比例、火候标准、摆盘规范。Skills 就是你给 Claude 写的 SOP。 可以构建 Skills 的任务几乎没有边界：

  • 内容创作 - 品牌语气、文章结构、写作禁忌——每次写东西都调用
  • 研究与分析 - 你习惯的分析框架、输出格式、信息来源偏好
  • 语法与编辑 - 你不喜欢的句式、常见错误清单、审校标准
  • 外联与销售 - 邮件模板、信息提炼方式、个性化策略
  • 求职应聘 - 你的简历、经历亮点、自我定位叙述方式

💡 核心原则： 任何你会重复告诉 Claude 的事情，都值得变成一个 Skill。

四步构建你的第一个 Skill ​

[圖 3] 整个过程不超过 30 分钟。你只做一次，但之后每天都在收益。 步骤 1: 启用 Skill-Creator 进入 Claude 账户的 Customize → Skills，找到并开启「Skill-Creator」。 这是 Anthropic 官方预置的一个特殊 Skill，它的唯一功能是帮你构建其他 Skills。为什么要有这个东西？因为构建一个高质量的 Skill 文件本身是有门槛的——你需要知道怎么组织信息、怎么写规则、怎么给出示例、怎么设置自查清单。Skill-Creator 内置了这套知识，引导你一步步完成。

💡 类比： 用 Skill 造 Skill——有点像用机床制造机床。 步骤 2: 把你的需求告诉 Claude 启用 Skill-Creator 后，你可以直接说： “我想为【某项工作流】构建一个 Skill，帮我构建它。” 但如果你想构建一个真正高质量的 Skill，最好用一个结构化的提示词，把上下文尽量完整地给到 Claude：

plaintext
你正在构建一个 Claude Skill——以 Markdown 格式保存的可复用指令集。

我的 Skill 用于：[描述任务，例如写 B2B 博客文章、审校语法、撰写销售邮件]

以下是 Claude 完成此任务需要的上下文：
- 我的名字/品牌：[你的名字或公司名]
- 我的受众：[你在为谁写作/服务]
- 我的语气和风格：[正式/非正式、专业/亲切、简洁/详细……]
- 我的输出标准：[好的输出是什么样的，越具体越好]
- 需要避免的内容：[禁忌词汇、格式要求、常见错误……]

请用上述上下文，编写一个完整的 SKILL.md 文件，要求：
1. 以一行文字说明此 Skill 的功能
2. 定义 Skill 激活时 Claude 扮演的角色
3. 列出 Claude 必须遵守的具体规则（越具体越好）
4. 至少包含一个优质输出的真实示例
5. 以 Claude 在输出前自查的质量清单结尾（3-5 个问题）

格式为标准 Markdown 文件，可直接保存并上传至 Claude Project。

⚠️ 关键细节： 你提供的上下文越具体，Skill 就越好用。笼统的“语气要专业”，不如“语气参考 Paul Graham 的文章风格：直接、不用废话、句子短”。 步骤 3: 保存到账户 Claude 构建完毕后，会输出一个 Markdown 文件（有时是 zip 包）。界面上有「Copy to Skills」按钮，点击即可保存到你的 Anthropic 账户。 同时强烈建议在本地建一个专用文件夹（比如 ~/Documents/Claude-Skills/），把所有 Skill 的 .md 文件都备份在里面。原因有两个：

  • 方便日后手动编辑迭代
  • 账户如果出问题，本地备份是你唯一的保障 一个花了几小时打磨出来的 Skill 文件，值得被认真对待。 步骤 4: 调用 回到 Customize → Skills → My Skills，确认新建的 Skill 已经处于启用状态。 之后在任何 Claude 对话里，只需要说：
  • “用我的品牌音调 Skill，写一封【主题】的通知邮件”
  • “用我的语法检查 Skill 审校这段文字”
  • “用我的研究 Skill，整理关于【话题】的核心观点” Skill 会被自动识别并加载，Claude 会按照文件里的所有指令行动。从此，你不需要再解释你是谁，不需要重复你的标准，不需要粘贴任何背景。

如何构建真正好用的 Skill ​

[圖 4] 这是整篇文章最核心的部分。任何人都能构建一个 Skill，但一个真正好用的 Skill 和一个平庸的 Skill 之间，差距可以非常大。 💡 技巧 1: 反向提问，让 Claude 来引导你 大多数人一坐下来就开始往提示词里填内容，然后发现自己不知道该填什么——我的语气是什么？我的受众是谁？我的禁忌词有哪些？ 换一个角度： 你不需要自己想清楚该给什么信息。告诉 Claude：

“我想构建一个 [X] Skill，请问我 10 到 50 个问题，帮助你更好地构建它。” Claude 会问你所有应该被问到的问题。这些问题本身就是一份清单——告诉你构建一个高质量 Skill 需要哪些信息。回答完这些问题，90% 的工作就完成了。 🔍 技巧 2: 反向发现，找你从没想到的场景 Skills 的另一个强大用法，不是解决你已知的重复问题，而是发现你没意识到自己在做重复事情的地方。 告诉 Claude: “基于你对我的了解，哪些工作流对我来说最适合构建成 Skill？” 如果你和 Claude 已经有过一段时间的对话历史，它对你的工作模式其实了解颇深，能给出相当有价值的建议。作者用这个方法发现了几个自己根本没想到的 Skill 应用场景。 🔄 技巧 3: 把现有对话直接转化成 Skill 这是整个 Skills 体系里最被低估的一个技巧。 如果你之前和 Claude 就某个话题深聊过很多次，那段对话记录本身就是最好的 Skill 原料。你不需要从头描述你的偏好——那些偏好已经隐藏在你们交互的每一条记录里。直接说： “把我们这段对话里的一切，转化成一个 Skill 文件。” 真实案例： 作者的语法检查 Skill 就是这样来的。他和 Claude 就语法偏好聊了几个月，积累了大量具体的反馈和细节。当他用这段对话生成 Skill 时，Claude 已经完整“理解”了他所有的偏好，一步到位生成了一个非常精准的文件。 ⭐ 技巧 4: 贴一个真实范例，胜过一百条规则 这是提升 Skill 质量最快的单一动作。 与其花时间写“语气要自然”“避免陈词滥调”“结构要清晰”这类泛泛的规则，不如把一篇你觉得真正完美的文章、一封你觉得极好的邮件，直接粘贴进 Skill 文件，附上一句话说明： “这是我想要的输出质量，以此为标准。” Claude 从一个具体的好例子里获取的信息，远比从十条描述性规则里获取的多。而且，你只有在看到真实输出的时候，才能真正说清楚“好”意味着什么——抽象描述往往是在偷懒。 🔁 技巧 5: 把第一稿当草稿，迭代三轮才满意 Claude 生成的第一版 Skill 通常是一个不错的起点，但不是终点。作者说他通常需要 三轮迭代 才会真正满意。 方法很简单： 认真读完第一版，把所有你觉得不对劲、太笼统、缺了什么的地方写下来，然后告诉 Claude 修改。第二版读完，继续挑问题，继续改。这个过程费时间，但不费精力——你只是在审阅和指出问题，Claude 来完成修改。 ✅ 技巧 6: 在 Skill 结尾加一个自查清单 每个 Skill 文件的最后，放三到五个 Claude 在生成输出前要问自己的问题：

  • 这段输出符合指定的语气要求吗？
  • 有没有出现禁忌词汇？
  • 格式是否符合标准？
  • 这个输出达到了示例的质量水平吗？ 这几个问题看起来微不足道，但它们会在每次输出之前强迫 Claude 做一次对照检查，相当于把你的质控流程内化进了 Skill 本身。 📖 技巧 7: 手动通读 Skill 文件 这是最枯燥的一步，但作者明确说“无论如何都要做”。拿出几分钟，从头到尾认真读一遍你的 Skill 文件。你会发现一些问题：
  • 某条规则其实含糊不清
  • 某段描述有歧义
  • 某个细节 Claude 其实理解不了 这些东西在你“试用”Skill 的时候不一定会暴露，但在你手动阅读的时候会变得显而易见。

💎 价值： 这几分钟的人工投入，换来的是之后每次调用都不出错。

Claude Skills 2.0：三个在 2026 年 3 月上线的重要升级 ​

[圖 5] Anthropic 在这次更新里解决了三个长期以来困扰重度用户的核心问题。 1️⃣ 内置评估与测试：你终于能知道 Skill 是否真的在工作 这是三个更新里最实质性的一个。 之前的痛点：你构建了一个 Skill，用了几次，感觉还不错，就认为它在正常工作了。但“感觉不错”是非常不可靠的标准。如果某次模型更新微调了 Claude 的输出倾向，你的 Skill 可能已经失效了——但你根本不知道，直到某天一个实际交付物出了问题。 现在的解决方案： Skills 2.0 引入了在保存前测试的机制。你写几个真实场景下会用到的提示词，然后 Claude 用这些提示词运行两次——一次激活你的 Skill，一次不激活。两次输出都会按照你预先定义的 断言（assertions） 打分，然后并排显示在预览界面里，供你对比和评价。

📌 什么是“断言”？ 你需要告诉系统什么叫“好输出”——比如“输出不能超过 200 字”、“必须包含数据来源”、“语气应该是对话式的”。系统会自动评估输出是否满足这些条件，并给出分数。 这意味着你在部署 Skill 之前，就已经知道它会产生什么样的结果。从“凭感觉”到“有依据”，是一个根本性的转变。 2️⃣ A/B 测试：改进 Skill 不再靠猜 你对一个 Skill 做了修改，加了一条规则，调整了一个措辞，改写了示例——你怎么知道这些改动真的有效？ 以前的方法：改完之后用一段时间，如果感觉变好了就留下，如果感觉没变就继续改。这基本上是在靠主观印象做决策。 现在你可以做 A/B 测试： 把原版和修改版对着相同的一组提示词各跑一遍，系统会把两个版本的输出并排呈现，让你直接对比。 💎 价值： 这对于维护长期使用的 Skill 来说尤其有价值。当你不确定某次改动是否真的提升了质量，测试数据会给你一个明确的答案。改版不再靠感觉，靠实验。 3️⃣ 触发词优化：解决“Skill 根本不响应”的问题 Skill 有一个经常被忽视但很关键的机制：触发。 你说“用我的品牌 Skill 写一篇帖子”，Claude 需要识别这句话里的意图，然后找到并加载对应的 Skill。如果你的 Skill 描述写得不够清晰，或者措辞和你的调用方式不匹配，Skill 就不会被触发——整个文件等于沉默地躺在那里。 这个问题比听起来更常见，尤其是当你有多个 Skill 的时候。Skills 2.0 引入了 自动触发优化：系统会自动重写并测试不同版本的 Skill 描述，测量每个版本在各种调用措辞下的触发成功率，最终找到一个最可靠的版本，整个过程自动完成。 使用方式： 在加载了目标 Skill 的 Claude 对话里，直接说：

  • “对这个 Skill 做评估”
  • “帮我 A/B 测试这两个版本”
  • “优化这个 Skill 的触发描述”

真实工作流：七种已经验证的 Skill 类型 ​

[圖 6] 作者在文章里展示了三种最常用的核心 Skill，并列出了更多值得构建的方向 📝 品牌音调 Skill 这是最基础、也最常被提到的一种。 包含内容：

  • 品牌的核心价值主张
  • 目标受众画像
  • 语气描述（用具体形容词，不用“专业”这种泛词）
  • 禁忌词汇清单
  • 几篇实际输出的范例
  • 不同内容类型（博客/社媒/邮件）的特定规则 建好之后，每次写任何类型的内容都先调用它。你的所有输出会自然具备一致的品牌声音，不需要每次自己把控，也不需要校对修正。 📄 PDF 生成器 Skill 输入任何文本，输出格式整洁、结构清晰的 PDF。 包含内容：
  • 布局规范（页眉页脚、字体大小、间距）
  • 内容组织方式（是否需要目录、章节格式）
  • 视觉风格（颜色、边距） 适合需要频繁生产报告、简报、提案的工作场景。 📋 文件摘要 Skill 粘贴任何文本，输出结构化摘要。不同于通用的“帮我总结一下”，一个专门的摘要 Skill 可以：
  • 定义输出的格式（要点列表还是段落）
  • 定义必须保留的信息类型（数据、结论、行动项）
  • 定义长度控制规则

🎯 关键： 一致性是关键——每次摘要都按同一套标准输出。 🎓 ELI5 Skill（Explain Like I'm 5） 把任何复杂话题翻译成普通人能理解的语言。粘贴一篇学术论文的摘要或者技术文档，触发 Skill，得到一段可以直接分享给非专业受众的解释。 Skill 里可以定义：

  • 类比风格（日常生活类比优先）
  • 目标读者背景（“假设读者没有任何相关领域知识”） 💼 求职 Skill 这是个人效率场景里 ROI 最高的 Skill 之一。 预存内容：
  • 完整简历
  • 主要项目经历
  • 核心技能描述
  • 你对自己最有说服力的叙述方式 之后每次申请新职位，只需要提供职位描述，Claude 就能自动生成一封针对性极强的求职信——不是泛泛的模板，而是真正根据你的具体经历和对方的具体需求量身匹配的内容。 📚 学习 Skill 你吸收新知识有自己的方式：
  • 有人需要类比

  • 你的目标（增肌/减脂/提高耐力）

问题还是那一个 ​

研究摘要 ​

TL;DR：Claude Skills 從入門到精通實戰：1 個 MARKDOWN code block 含完整 skill 結構模板（frontmatter + 觸發條件 + 執行步驟 + 輸出格式）

核心論點 ​

  • Skills 是 Claude Code 的「肌肉記憶」：技能讓重複性工作流一次定義、永久複用
  • Skills 設計原則：明確的觸發詞（trigger）；確定的輸出形狀（output contract）；原子化步驟（每步可驗證）
  • 進階技巧：Skills 可以呼叫 Skills（巢狀技能）；Skills 可以有參數；使用 @file 引用外部文件
  • 完整 Skill 結構模板：1 個 MARKDOWN code block 示範標準 frontmatter + 執行步驟格式

關鍵數據 / 工具 ​

  • 1 個 MARKDOWN code block（完整 skill 結構模板）
  • Skill 命名慣例：verb-noun 格式（如 code-review / deploy-app）

  • 1 個 code block 偏少：文章主題是 Skills 實戰，但只有 1 個具體代碼示例
