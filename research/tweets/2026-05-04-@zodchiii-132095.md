# The Claude Skills Folder That Automates 90% of Workflow

**Original URL**: https://x.com/zodchiii/status/2051227992404132095

## TL;DR

Practical guide to creating custom slash commands in Claude Code via `.claude/skills/` folder; provides 10 templates, YAML mechanics, and a 9-step automation workflow.

## Full Article Text

Claude Code has 60+ built-in slash commands. /compact, /clear, /model, /review. Most people know 5-6 of them.

What most people don't know is you can create your own. A markdown file in .claude/skills/ becomes a slash command you invoke with /name. Write the prompt once, use it forever.

Copy the folder, drop it in your project, and your daily workflow gets faster today.

Here's the full setup 👇

### How custom commands work

Create a markdown file with YAML frontmatter. Put it in one of two places:

The file name becomes the slash command. review/SKILL.md becomes /review. That's it.

The frontmatter controls behavior:

- description tells Claude when to auto-invoke the skill. 
- allowed-tools limits what the command can do. 
- Everything below the frontmatter is the prompt Claude follows.

You can pass arguments too. $ARGUMENTS captures everything after the command name, and $1, $2 capture positional args.

### commands/ vs skills/ — which one?

Quick answer: use skills/. As of April 2026, Anthropic merged custom commands into the skills system. Both work, but skills add two things commands don't:

1. Skills can be triggered automatically based on context. If your /review skill description says "use when reviewing pull requests or code changes," Claude will suggest it when you're working on a PR even if you don't type /review.
2. Your existing .claude/commands/ files keep working. No migration needed, but new commands should go in .claude/skills/.

### The 10 commands (copy-paste ready)

1. **/review** — Code Review
2. **/test** — Write Tests
3. **/commit** — Smart Commits
4. **/pr** — Create Pull Request
5. **/debug** — Investigate Errors
6. **/refactor** — Safe Refactoring
7. **/docs** — Generate Documentation
8. **/migrate** — Database Migration
9. **/deploy-check** — Pre-Deployment Verification
10. **/security** — Security Scan

### How to install all 10

Copy this folder structure into your project or globally (~/.claude/skills/). Commit the project-level ones to git for team-wide access.

### The workflow that brings it all together

A typical feature cycle using these commands:

1. /debug         → investigate the issue
2. Write code     → implement the fix
3. /test          → generate tests for new code
4. /refactor      → clean up if needed
5. /review        → self-review before PR
6. /security      → quick security scan
7. /commit        → create structured commits
8. /pr            → generate PR description
9. /deploy-check  → verify everything before ship

Each step that used to be a 50-word prompt is now 7-15 characters. Over a month, that compounds into hours saved.
