你有没有过这种经历：让 Claude Code 做某件事，它就是不做？

你说"格式化代码"——它没做。你说"别碰那个文件"——它碰了。

你说"完成前跑一下测试"——它忘了。

原因在于，CLAUDE.md 只是一个建议。

Claude 读了它，大概 80% 的时间会遵守。但 Hooks 不同。它们是自动触发的动作——每当 Claude 编辑文件、执行命令或完成任务时，都会自动执行。

下面 8 个 Hooks，直接复制到 settings.json，设完就忘。

 

30 秒搞懂原理

Hooks = 绑在 Claude Code 动作上的脚本。设一次，后台一直跑。

PreToolUse：Claude 动手之前。exit code 2 = 拦住。门卫。

PostToolUse：Claude 动手之后。跑格式化、跑测试。质检员。

 

文档：https://code.claude.com/docs/en/hooks

## 研究摘要

**TL;DR**：8 個 Claude Code Hooks 的實戰腳本全集，附完整 bash 代碼。覆蓋：自動格式化、阻擋危險命令、保護敏感文件、改後跑測試、建 PR 前強制測試通過、自動 lint、審計日誌、自動 commit。直接可複製到 `.claude/settings.json` 的完整 JSON 配置。

### 核心論點
1. **Hooks 解決「Claude 說搞定但沒測試」問題**：PostToolUse hook 自動跑測試，`tail -5` 只留最後幾行防止 context 爆掉
2. **安全閘道是核心防護**：`rm -rf`、`DROP TABLE`、`git push --force` 等危險命令用 exit code 2 阻斷（PreToolUse hook）
3. **敏感文件保護**：`.env*`、`*.pem`、`secrets/*` 路徑偵測 + exit 2，防止 Claude 無意修改
4. **建 PR 前強制測試通過**：獨立腳本在 PreToolUse（Bash，matcher 含 `gh pr create`）時執行 `npm run test`，失敗則阻斷
5. **審計日誌**：每個 Bash 命令記錄到 `.claude/command-log.txt` 加時間戳，出事有 trace
6. **自動 commit 有爭議**：`Stop` hook 觸發 `git commit -m "chore(ai): apply Claude edit"`，搭配 worktrees 每任務一分支

### 關鍵數據 / 工具
- Boris Cherny 引言：「實時反饋能讓輸出質量翻 2-3 倍」
- Hook events 使用：PostToolUse（Edit trigger → 跑測試/lint）、PreToolUse（Bash trigger → 安全檢查）、Stop（完成時 auto-commit）
- 腳本路徑：`.claude/hooks/` + `chmod +x *.sh`
- 完整 settings.json 範例有在文章中（含所有 8 個 hooks）
