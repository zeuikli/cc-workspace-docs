## 研究摘要

**TL;DR**：介紹 phuryn/claude-usage 工具，可解析 Claude Code 本地 JSONL 日誌並生成本地 dashboard，顯示 token 用量、成本、模型分布等詳細數據。3 分鐘安裝，純 Python 標準庫，零依賴，開源 MIT。

### 核心論點
1. **官方缺乏透明度**：Claude 官方只提供進度條，不顯示 token 明細，用戶無法知道哪個專案在消耗資源
2. **本地日誌已存在**：Claude Code 本地寫了完整的 JSONL 日誌（~/.claude/projects/），phuryn/claude-usage 只是把它可視化
3. **數據驅動優化**：看到明細後才能針對性優化 — 「知道哪個專案在偷吃 token，及時優化 prompt 或換模型」
4. **安裝簡單**：git clone + `python3 cli.py dashboard` 即可，自動掃描歷史 + 增量更新

### 關鍵數據 / 工具
- 工具：phuryn/claude-usage (github.com/phuryn/claude-usage)
- 當時星數：419（剛起火，現應更高）
- 安裝命令：`git clone https://github.com/phuryn/claude-usage.git && python3 cli.py dashboard`
- 額外命令：`python3 cli.py today` / `stats` / `scan`
- 作者揭露自己的資料：1 週重構專案消耗 180 萬 token，按 API 計算約 500 人民幣
- 378 likes · 595 bookmarks · 43,707 views

### 批判性觀察
- **Max 計劃用戶最受益**：固定月費計劃下不需要擔心成本但需要監控用量上限；API 用戶更需要精確成本控制
- 這個工具是「被動監控」，不會自動減少 token，需要用戶主動根據數據調整
- 聲稱「已省 2000+ 元」是誇大：工具只是顯示數據，省錢是用戶自己的行為改變
- 「1 週重構消耗 180 萬 token，如果按 API 算要 500 元」此比較不太準確（Max 計劃費用不等於 API 費用）
